Shadow routing + Ultimate Policy Gradient

OpenClawBrain v12.2.6 · March 2026 · Jonathan Gu

OpenClawBrain now prioritizes one operating rule: keep query-time retrieval fast, then learn better routing asynchronously.

Hot path stays local: 
embed(query) -> local traversal -> prompt_context -> answer

No teacher LLM call runs on this path.

Background learning loop: 
sample recent route decisions
-> ask teacher (gpt-5-mini) what it would choose
-> apply policy-gradient updates to edge weights + relevance metadata
-> feed improved signals into split/merge/prune/connect

Ultimate Policy Gradient (the central claim)

All routing supervision is turned into the same kind of policy update. The only difference is weight and volume:

  • Human feedback: highest authority, highest weight per sample.
  • Self-learning outcomes: strong signal (works even when the teacher is off).
  • Harvester signals: medium strength (coverage across many conversations).
  • Async teacher labels: lowest weight per label, but massive volume via replay.

Why this shift matters

  • Better routing quality without adding hot-path latency.
  • Fewer turns and fewer prompt tokens on repeated workflows.
  • Cleaner context blocks because weak routes are gradually down-weighted.
  • Structural learning remains core: split/merge/prune/connect + injection, now fed by cleaner routing signals.
This is a design update and operational architecture statement. Claims of benchmark improvements should be supported by measured evals.

How to run

# Dry-run by default (no writes)
openclawbrain async-route-pg --state /tmp/brain/state.json --json

# Apply updates (writes state)
openclawbrain async-route-pg --state /tmp/brain/state.json --apply --json

Dry-run first, inspect proposed route updates, then apply.
