OpenClawBrain is the memory layer for OpenClaw: it keeps corrections, picks cleaner context for each job, and gets sharper on work that comes back around. The default direction is simple: one local brain per machine, retrieval chosen per ask, and learning off the hot path. This blog is the map. Start with install and proof, then go deeper into the architecture once you want to know how the guide actually thinks.

The operator path is now real enough to document plainly. Install provisions the default bge-large embedder before brain init unless you explicitly skip it. Eagle and Tern both attached on real profiles; Tern proved EXTERNAL RESTART on a Telegram-connected profile; Eagle proved learner start, first export, promoted-pack serving, learned route_fn, and graph truth updates on later served turns, with live bge-large embeddings and an enabled local qwen3.5:9b teacher on that machine. A shared machine brain is acceptable when you want one memory surface, while dedicated roots still buy the clearest rollback, disable, cleanup, or uninstall boundary.

Concepts

Why agents forget — and how learning fixes it
Concept
Every conversation starts from scratch. OpenClawBrain builds a persistent map of what your agent knows and updates it in the background from corrections and outcomes — so the next conversation starts smarter.
Context selection vs. keyword search
Concept
RAG finds text that looks similar. OpenClawBrain blends two signals — what the knowledge map knows about how facts connect, and what fits this specific question — so the right context can surface even when the wording does not match.
Learning from corrections
Concept
When your agent makes a mistake and you correct it, OpenClawBrain records what went wrong and updates its routing weights off the live path. The agent keeps running; the guide gets better for the next round.
OpenClaw vs. OpenClawBrain
Concept
OpenClaw is the runtime and orchestration layer. OpenClawBrain is the memory and context-selection layer behind it. If the brain is unavailable, the agent keeps working; fail-open is part of the design, not a footnote.
