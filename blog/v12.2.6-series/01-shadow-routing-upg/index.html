Post 1: Shadow Routing + Ultimate Policy Gradient

March 2026 · v12.2.6+ series

The current OpenClawBrain design separates runtime routing from heavy learning. Runtime is local and bounded. Learning happens asynchronously over replay data.

Shadow Routing

"Shadow routing" means teacher models do not sit on the query hot path. They run later, label prior decisions, and produce updates for the next deployment window.

Hot path: embedding lookup -> local route policy -> bounded context output.
Cold path: replay -> teacher labels -> distillation + policy-gradient updates.

Ultimate Policy Gradient

All supervision enters one update stack with an explicit authority order:

  1. Human feedback
  2. Self-learning outcomes
  3. Harvested weak signals
  4. Async teacher labels

The hierarchy ensures high-authority corrections can override volume-driven but weaker labels.
