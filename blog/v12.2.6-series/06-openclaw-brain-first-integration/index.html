Post 6: OpenClaw integration boundary

This route now mirrors the public repo more closely: OpenClaw keeps the hot path, OpenClawBrain compiles from the active promoted pack, and learner refresh changes later served turns only after promotion.

Integration contract

  • OpenClaw owns session flow, prompt assembly, response delivery, and fail-open serving.
  • OpenClawBrain supplies normalized events, immutable pack materialization, activation staging/promotion, deterministic compile, and learned-route diagnostics.
  • The current code does not prove in-place mutation of the active pack during serving.

Exact landing shape

  1. compile from activation’s active pointer
  2. emit compile diagnostics from the promoted pack
  3. export normalized events off-path
  4. refresh candidate packs later
  5. serve fresher routing only after promotion

Install and rollout truth now

The current lane is repo tip plus local tarballs, with attach, observability, and rollback checks from the public repo docs. Do not imply bare npm publication or shadow/live rollout proof yet.
