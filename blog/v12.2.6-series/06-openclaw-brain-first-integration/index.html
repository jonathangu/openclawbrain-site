Post 6

Brain-first OpenClaw integration

March 2026 | v12.2.6+ series

Brain-first mode is the recommended way to use OpenClawBrain because it matches the real product story: OpenClaw can use the brain immediately, and the brain can keep learning in the background without changing the live request path into a training job.

Before and after

Before brain-first

user message
  -> OpenClaw loads prompt + files
  -> model answers
  -> optional manual memory updates later

After brain-first

user message
  -> query_brain(summary, chat_id)
  -> learned route_fn returns bounded [BRAIN_CONTEXT]
  -> model answers with brain context
  -> capture feedback / learn_by_chat_id
  -> replay + harvest + async teacher improve later turns

What changes operationally

  • OpenClaw gets a memory-routing step before answer generation.
  • Corrections target the actual fired route through the same chat_id.
  • Historical replay and async teacher labeling improve the next runtime router, not the current turn.

Fail-open is part of the design

If the brain query path fails, OpenClaw should continue normally. That gives you a clean rollout and rollback path.

  • keep prompt output in bounded [BRAIN_CONTEXT]
  • keep --exclude-bootstrap enabled to avoid duplication
  • do not inject raw retrieval internals into the prompt
  • do not put the async teacher in the live request path

Enablement path

openclawbrain serve --state ~/.openclawbrain/main/state.json
openclaw hooks install /path/to/openclawbrain/integrations/openclaw/hooks/openclawbrain-context-injector
openclaw hooks enable openclawbrain-context-injector
openclaw gateway restart

If the hook pack is not available yet, use the adapter commands directly in AGENTS.md.

Continuous learning is the point

Once the brain is in the loop, keep the background path running:

  1. replay historical and ongoing sessions
  2. harvest scanner labels
  3. apply async-teacher labels
  4. update the runtime route policy with Ultimate Policy Gradient

That is how OpenClawBrain becomes more valuable over time without slowing down the live OpenClaw turn.
