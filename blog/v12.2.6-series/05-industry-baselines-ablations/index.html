Post 5: Industry Baselines + Ablation Study

March 2026 · v12.2.6+ series

We align our evaluation with industry standard baselines so it is easy to compare routing systems on real operator workloads. This post explains why those baselines exist, how we map them to OpenClawBrain modes, and which metrics we report.

Why These Baselines

BaselineWhat It RepresentsWhy It Matters
Vector top-k Plain embedding retrieval without traversal. Cold-start floor: strong when there is no learned routing yet.
Top-k + reranker Deterministic re-ranking over candidate edges (edge+sim). Represents classic “cheap rerank” stacks without learning.
Pointer-chasing Iterative multi-hop traversal without learned routing policy. Captures the common failure mode of repeated pointer-following in memory systems.

Mapping to harness modes:

  • Vector top-k → vector_only
  • Top-k + reranker → edge_sim_legacy (deterministic route_mode=edge+sim)
  • Pointer-chasing → edge_sim_legacy with multi-hop traversal enabled (the harness default)

OpenClawBrain learned routing is reported as learned, with graph_prior_only and qtsim_only as ablation bounds.

How To Reproduce

Use the canonical repro steps in docs/reproduce-eval.md. The short version:

cd /path/to/openclawbrain
python examples/eval/run_eval.py \
  --state /path/to/state.json \
  --queries /path/to/queries.jsonl \
  --modes vector_only,edge_sim_legacy,graph_prior_only,qtsim_only,learned \
  --output /tmp/ocb_eval.json \
  --print-per-query

python examples/eval/simulate_expert_regions.py --output-dir /tmp/ocb_expert_regions
python examples/eval/simulate_two_cluster_routing.py --output-dir /tmp/ocb_two_cluster

python3 /path/to/openclawbrain-site/scripts/plot_eval.py \
  --inputs /tmp/ocb_expert_regions /tmp/ocb_two_cluster \
  --out figures/eval

Expected outputs include /tmp/ocb_eval.json, /tmp/ocb_expert_regions/simulation_curve.csv, and regenerated SVGs in figures/eval/.
