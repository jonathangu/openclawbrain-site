Post 4: Operator Guide for Local Embeddings OOTB

March 2026 · v12.2.6+ series

Local embeddings out-of-the-box remain the default routing signal on the live request path (the path the user waits on). In the shipped TypeScript-first workspace, boot starts from existing files immediately. Meanwhile, always-on learning keeps building improved brain packs — versioned bundles of context and routing weights — in the background.

Package-first Baseline Flow

corepack enable
pnpm install
pnpm check
pnpm release:pack

Operational Notes

  • Keep the learned routing function deterministic and bounded on the live request path. The compiler always produces the same output from the same pack, so rollback is safe and predictable.
  • Leave human labels, self-label harvesting, and scanner ingestion default-on so new signals land continuously.
  • Run continuous graph learning (connect/split/merge/decay/co-firing) in background loops, not inline with user requests.
  • Brain packs are immutable once built. You can switch which pack is active or roll back to a previous one instantly — no data-shape rewrites needed.
  • Treat mechanism proof and product proof separately: routing-mechanism wins do not automatically prove end-to-end operator outcomes.

Runtime ownership stays split: OpenClaw runs channels, sessions, prompt assembly, and fail-open control; OpenClawBrain runs packs, learning, and compile logic.
