Post 4

Works out of the box, no cloud required

March 2026 | v12.2.6+ series

Most AI memory tools need you to sign up for a vector database or send your data to someone else's API. OpenClawBrain doesn't. It runs entirely on your machine — local embeddings, local learning, no external calls — like a librarian who lives in your office instead of a call center. Four commands and the guide is serving context.

corepack enable
pnpm install
pnpm check
pnpm release:pack

What "local-first" means in practice

  • No API keys for the brain itself. The default stack uses local BGE-large embeddings and a local Ollama/Qwen teacher. Your broader OpenClaw agent may still need its own provider keys, but the brain doesn't.
  • Fast startup from existing files. Boot starts from what's already on disk — no cold-start wait. The guide is useful before any background learning finishes.
  • Learning happens in the background. Corrections, automated signals, and teacher labels continuously improve the guide — but never block a live response.

Operational guardrails

  • Deterministic and rollback-safe: The compiler always produces the same output from the same knowledge snapshot, so rollback is like swapping a book on a shelf — predictable and instant.
  • Immutable snapshots: Once built, a knowledge snapshot never changes. Switch to a previous one or promote a new one without data migration.
  • Feedback on by default: Human labels, self-label harvesting, and automated signals are default-on so the guide keeps improving without extra configuration.
  • Mechanism proof vs product proof: Routing-mechanism wins in benchmarks don't automatically prove end-to-end operator outcomes — we keep these separate.

The runtime split stays clean: OpenClaw (front of house) runs channels, sessions, prompt assembly, and fail-open control. OpenClawBrain (the kitchen) runs knowledge snapshots, learning, and compilation.
