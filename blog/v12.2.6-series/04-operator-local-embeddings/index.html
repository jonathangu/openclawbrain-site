Post 4: local-first attach posture

This route now stays close to the public repo: repo tip plus tarballs, promoted-pack compile, activation visibility, observability, and fail-open discipline. It no longer invents an operator config surface the repo does not publish.

Current operator truth

  • start from the repo tip and local .release/*.tgz tarballs
  • use the attach and observability docs in the public repo for the full operational path
  • treat active/candidate/previous state as explicit activation lineage
  • keep compile bounded and fail-open semantics explicit

Canonical docs

  • setup guide
  • openclaw integration
  • openclaw-attach-quickstart.md
  • operator-observability.md

Not implied here

  • no claim of a finished public operator YAML or JSON config schema
  • no claim that local embeddings alone define the whole product surface
  • no claim that graph maintenance proves live runtime plasticity
