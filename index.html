For OpenClaw operators

Your retrieval doesn't learn from outcomes.
OpenClawBrain fixes that.

Embedding similarity picks chunks per query with no memory of what worked before. OpenClawBrain adds a learned graph router that reinforces routes that succeed and suppresses routes that fail — inside your existing OpenClaw runtime.

How it works

Fast start. Continuous learning. Clean boundary.

  • Boot from existing files — no full pre-scan gate before the first useful route.
  • Learned routing on the hot path — the route function runs inline with bounded compute. Background teacher updates stay off the hot path.
  • Default-on labels — human corrections, self labels, and scanner/harvester signals all feed one learning loop.

The problem

Standard retrieval does not learn from outcomes.

Pain Fix in OpenClawBrain
Cold starts blocked by full-history prep. Boot from current files immediately; learn history in the background.
Label collection is inconsistent and optional. Human, self, and harvester labels run as default inputs.
Learning logic and runtime are tightly coupled. OpenClaw owns runtime; OpenClawBrain packages own contracts, provenance, and learner.
