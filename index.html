Why do you need this?

Your AI coding agent keeps making the same mistakes. You correct it, it forgets. You teach it a pattern, it's gone tomorrow. That's not intelligence - that's expensive forgetfulness.

Corrections stick forever

Use brain_teach once, and the brain surfaces that correction as priority context every time.

Learns from outcomes

Deploy fails? Test passes? The brain strengthens edges that led to good outcomes, weakens bad ones.

Smart context selection

Instead of dumping everything, the brain retrieves exactly what's relevant for this specific query.

Full transparency

Every decision traced. brain_trace shows seed choice, hops, fired nodes, and vetos.

How it works

1. Install

openclaw plugins install @jonathangu/openclawbrain

2. Initialize

openclawbrain init /path/to/workspace

Discovers your files, computes embeddings, builds graph, promotes first pack.

3. Teach

brain_teach instruction="Use gh not hub for PRs"

That's it. The brain remembers forever.

What you get

  • Lossless transcript memory - every conversation persisted
  • Learned routing graph - knows which context to surface
  • Immediate corrections - brain_teach works instantly
  • Paper-faithful RL - REINFORCE over full trajectories (Gu 2016)
  • Replay-gated packs - no regressions, mutations tested first
  • Fail-open design - serving continues if worker dies
