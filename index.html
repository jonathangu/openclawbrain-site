Useful memory for OpenClaw

Give OpenClaw memory that can help.

OpenClawBrain is an open-source memory layer for OpenClaw. It learns from surfaced history, notes, and corrections, then brings back a small useful slice when it matters.

The payoff is simple: fewer repeated fixes, less prompt bloat, and a better chance that yesterday's lesson is still available tomorrow.

In Plain English

It is built to carry forward useful lessons.

1. Learn from work you already did

Notes, surfaced history, docs, and explicit corrections can become memory material.

2. Keep live runs bounded

Learning happens away from the live prompt. The runtime serves a small slice instead of your whole archive.

3. Make it inspectable

You can check install, status, proof, and source instead of guessing what the system is doing.

Why Should I Care?

If you use an agent seriously, you end up teaching the same lesson twice.

That gets expensive fast. You fix a workflow, correct an assumption, or explain a preference, then you have to do it again next week because the useful part fell back into old scrollback. OpenClawBrain is meant to give those lessons a path back into the next run.

Less repetition

Stop restating the same fix

When you correct a workflow or preference, the useful outcome is seeing that correction before the same mistake repeats.

More continuity

Carry forward what mattered

Useful past work can stay available across sessions instead of every run starting from zero again.

Lean prompts

Keep the live prompt small

A bounded memory slice is easier to use than stuffing full scrollback into every prompt and hoping the right line wins.

Why this matters: memory only helps if it improves the next answer without making every prompt bigger and slower.

Why Is This Better Than Raw History Or Vanilla RAG?

The hard part is not storing text. It is deciding what should come back now.

Raw history is useful as an archive. Naive memory can save notes. Vanilla RAG can pull up similar text. None of those, by themselves, solve the live prompt problem: deciding what deserves to come back right now and keeping that answer small enough to use.

Raw history

Good archive, bad live context

A giant transcript remembers everything and helps with very little unless you manually sift it first.

Naive memory / vanilla RAG

Retrieval is not the same as learning

Similarity search can fetch related text, but it does not automatically learn which corrections actually improved behavior.

OpenClawBrain

Bring back a bounded useful slice

It learns from surfaced material, promotes what survives checks, and serves a small slice with learned selection instead of replaying the whole past.

The point: bring back the few things that improve the next answer, not more text for its own sake.

Why Should I Trust It?

You can inspect it instead of taking it on faith.

OpenClawBrain does not ask you to believe in invisible magic. The code is public. The install and upgrade lanes are documented. The live path is inspectable through status and proof instead of hidden behind marketing copy.

Open source Code, docs, releases

You can inspect the source, the install path, and the public record of changes.

Visible status status --detailed

There is a direct way to check whether the runtime is loaded and serving, not just installed on disk.

Bounded serving Small live slice

Learning happens outside the live prompt, while the prompt itself stays capped and usable.

Real install and upgrade path Single canonical install lane

New installs and upgrades use the same public lane instead of a private operator-only ritual.

Honest boundary: this is a bounded memory layer, not a promise of perfect recall. The value is that useful prior work has a better chance to change the next run, and you can inspect whether that path is active.

What Do I Do Next?

Install it on one OpenClaw home and verify it.

The install lane is intentionally short and remains the canonical path. Use the same selected home on every openclawbrain command, restart once, then check detailed status and proof.

Canonical install lane 
openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'

The restart command sits between install and verify, but it does not take the path flag itself.
