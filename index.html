One local brain for the agents you already use

Give the agents you use one source-backed memory.

The thing I keep coming back to is that agent memory should not be another confident summary with no source behind it. ocbrain keeps an append-only evidence and outcome ledger, derives a scoped current view, and gives each runtime only the coherent context its decision needs.

The store is local SQLite. A stable Shared Context packet gives every configured client the same retrieval shape, source handles expand exact material on demand, and closeout receipts connect actions and outcomes back to what the agent saw. Start with one compatible MCP client or connect several. OpenClaw is supported, not required.

Install first. Then make the proof real.

A good memory system is not installed when a config file looks plausible. It is installed when each client you chose can start the same MCP server, receive an ocbrain.context.v1 packet no larger than 32 KB, expand an issued source when one exists, record retrieval usefulness, and close out the task. The install guide starts with requirements and ends with those checks.

A fresh brain can prove a real source round trip.

A new database starts empty on purpose. v1.1.0 includes a tracked demonstration with four public installation facts and one hash-pinned source. Review it, then apply it with curated-apply --allow-hosted-egress. The acknowledgement covers only those exact fact bodies; it does not authorize the database, full source file, or a local filesystem path to leave the machine.

After that explicit step, a fresh Codex, Claude Code, or other hosted-model client can retrieve the public facts, receive an ocbrain:// source handle instead of a local path, and verify the source hash. Private or local_only material remains excluded before ranking and packet construction.

One brain does not mean one undifferentiated memory pool.

Every retrieval carries the narrowest context the runtime knows: project, repo, task, client, runtime, and session. That context is a lens over one ledger. Stable doctrine can travel; confidential or project-specific evidence does not.

Events are authority

History enters an append-only hash chain. A correction or tombstone is a later event, never a silent rewrite.

Belief is a projection

Evidence objects, current beliefs, aliases, provenance links, and search are rebuildable derived views.

Attention is bounded

FTS5 always works; an optional local Ollama sidecar can add dense candidates. Shared Context stays within 32 KB and exact sources expand only when needed.

Scope is mechanical

Scope and hosted-delivery policy are enforced before ranking. Get and source expansion are not ID-based bypasses.

Maintenance is explicit, local, and bounded.

The default install is an on-demand MCP process. It installs no recurring light, heavy, or stallcheck job. A bounded ocbrain sync reconciles the local event projection when an operator asks; it does not call a hosted judgment or embedding provider, build a dataset, train a model, or start a scheduler.

Hosted judgment, hosted embedding, training, and watchdog controls are absent from the core MCP. The optional dense path is loopback-only. ocbrain does not enqueue agent work, install skills, send hosted requests, train a model, or page an operator on its own.

Moving evidence is a deliberate file operation.

v1.0.1 adds manual cross-machine evidence bundles without turning sync into ambient authority. Export starts from an explicit evidence allowlist, applies scope and egress policy, redacts secrets, records an audit receipt, and writes a fresh owner-only file. Import validates first and touches no database unless the operator supplies --apply.

No network sync

The core does not discover peers, upload a corpus, or run a background replication service.

Evidence only

Sender IDs are provenance, not authority. Imports derive local IDs and cannot import beliefs, retrievals, closeouts, or companion state.

Private by arrival

Imported evidence is forced to confidential, local_only project scope until later local judgment says otherwise.

Not an MCP tool

Bundle exchange stays in the explicit operator CLI; an agent cannot initiate it through the eight-tool runtime profile.

The dataset is optional—and still the long game.

Local curation, grading, audit, and prepared training workflows now live in the separate ocbrain-training package and its own SQLite store. The core MCP neither imports nor queries it. The durable brain keeps changing facts and provenance; weights are reserved for stable voice, judgment, taste, and skills.

The second pilot improved and still failed the model-quality bar. It reused the first pilot's twenty prompts, references, rubric, held-out hashes, and blind randomization exactly. After eleven canonical first-party examples were added locally, the candidate improved from 2 preferences to 7; the reference still won 13.

The AI-delegated 150-item audit found 67 pass / 83 fail, so pilot-v3 remains blocked. Those rulings are remediation evidence, not named-human approval. A clean private Markdown handoff omits the AI labels; even a completed human review still requires remediation, reminting, regrading, a fresh stratified audit, and separate operator authorization before training.

The v1 migration is archive-first.

The migration command opens the live database read-only, preserves the exact event prefix, builds a strict event-authoritative core, extracts training and operations into separate stores, and publishes a verified archive plus manifest at fresh paths. It never converts in place.

The July 13 rehearsal produced all five artifacts and verified 671,580 events, 232,677 evidence objects, 137,530 beliefs, zero foreign-key violations, and matching manifest hashes. Migration did not activate or repoint a client. After the remaining gates passed, the operator provisionally pointed fresh clients at a byte-identical v1 live copy; Codex, Claude Code, and OpenClaw all completed the full round trip, so the pointer was retained.

Formerly OpenClawBrain.

The project began as OpenClawBrain and became the smaller ocbrain runtime: local first, source-backed, shared through MCP, and conservative about what earns memory. The domain stayed because the question stayed the same. How should an agent remember without turning arbitrary text into truth?
