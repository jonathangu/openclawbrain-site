The wedge

Standard retrieval does not learn from outcomes.

Embedding similarity selects chunks independently per query with no memory of what worked before. OpenClawBrain adds a learned graph router that improves with every interaction: routes that succeed get reinforced, routes that fail get suppressed.

Pain Fix in OpenClawBrain
Cold starts blocked by full-history prep before usable routing. Boot from existing files immediately, then learn historical context continuously in the background.
Label collection is inconsistent and often optional. Human labels, self labels, and scanner/harvester labels run as default inputs to learning.
Learning logic and runtime orchestration are tightly coupled. OpenClaw runtime stays separate while OpenClawBrain packages own contracts, provenance, packs, compiler, and learner behavior.

How it starts and keeps learning

Fast boot from existing files. Always-on learning in the background.

  • Immediate boot: runtime boots from existing workspace files without waiting for a full historical pre-processing gate. First useful routing is available immediately.
  • Background learning: the learner processes historical and incoming data continuously. New incoming data is prioritized first.
  • Default-on label hierarchy: human feedback (highest weight) → self labels → scanner/harvester labels → async teacher labels. All feed into one learner loop.
  • Hot/off-path split: the learned route function runs on the hot path with bounded compute. Async teacher updates and continuous graph learning (structural updates, decay, Hebbian co-firing) stay off the hot path.
