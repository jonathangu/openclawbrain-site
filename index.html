Lightweight shared memory for local agents

ocbrain gives Codex, Claude, and OpenClaw one source-backed memory layer.

The lightweight ocbrain runtime ingests historical OpenClaw, Codex, Claude, workspace, and task artifacts into a local SQLite ledger. It serves compact context through MCP, records whether returned memories helped, and keeps durable memory/wiki/skill/policy writes behind proposal gates.

The domain openclawbrain.ai now points at the smaller installed ocbrain path: shared local knowledge first, visible evidence, conservative mutation, and no silent rewriting of skills or policy.

Do not make each agent remember alone. Give them one evidence-backed memory layer and a feedback loop.

Runtime tools

brain.search

Search source-backed OpenClaw, Codex, Claude, task, and workspace history without making each runtime keep a separate memory silo.

brain.digest

Expose ledger counts and state so memory quality is inspectable instead of vibes-based.

brain.get

Retrieve reviewed candidates with scope and draft/private controls, instead of injecting raw transcripts blindly.

brain.feedback

Record helpful, used, ignored, irrelevant, or harmful outcomes against a retrieval-use id.

What ocbrain is, and why you can trust it

ocbrain is a local, source-backed consolidation brain for coding agents. Immutable evidence goes in; a controlled compiler turns that evidence into current knowledge; and memory is a read-only view over that knowledge. Nothing durable is invented in the dark — every belief traces back to something that actually happened.

Evidence before belief

Raw history is captured as immutable evidence first. Knowledge is never written ahead of the evidence that justifies it.

Knowledge is compiled

Beliefs are compiled from evidence, lifecycle-managed, and linked back to the evidence they came from — so any claim can be traced and re-checked.

Read-first at runtime

Runtime consumption is read-first and source-backed. Agents retrieve reviewed, attributed knowledge rather than injecting raw transcripts.

Human gate for risk

Executable, prescriptive, or high-risk knowledge is proposal-first: it requires an explicit human gate before it becomes durable.

Safety stance: ocbrain is a librarian and compiler with a security desk, not an autopilot.

Not an autopilot No loop-runner No scheduler No skill-installer Never widens privacy scope External content is data, not instructions Supersedes & archives, never deletes history

Quick start

git clone https://github.com/jonathangu/ocbrain.git
cd ocbrain
/opt/homebrew/bin/python3.13 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
ocbrain --db data/ocbrain.sqlite mcp
