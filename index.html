One local brain for the agents you already use

Let ChatGPT, Codex, Claude Code, and OpenClaw remember together.

The thing I keep coming back to is that agent memory should not be another confident summary with no source behind it. ocbrain reads the histories these runtimes already produce, compiles them into scoped knowledge, and serves only the parts that have earned trust.

The store is local SQLite. Evidence stays attached to belief. Corrections, tests, tool results, and retrieval feedback change what the brain promotes. The training corpus stays on the machine, so the same history can improve memory now and a personal model later.

Install first. Then make the proof real.

A good memory system is not installed when three config files look plausible. It is installed when each runtime can call the same MCP server, retrieve scoped context, and record whether that context helped. The install guide now uses the current runtime CLIs and ends with those checks.

One brain does not mean one undifferentiated memory pool.

Every retrieval carries the narrowest context the runtime knows: project, repo, task, client, runtime, and session. That context is a lens over one ledger. Stable doctrine can travel; confidential or project-specific evidence does not.

Evidence is immutable

History enters as hash-pinned evidence. The current belief can change without rewriting what happened.

Knowledge is compiled

Signals from corrections, verifiers, outcomes, and retrieval use decide what remains current.

Memory is a view

Agents see a bounded view over current injectable knowledge, not a second store that can drift.

Scope only tightens

A derived object inherits the most restrictive linked source scope. Private evidence cannot make a public belief.

The system maintains itself, but it does not act in the world.

The light profile keeps recent knowledge labeled, embedded, checked, promoted, and rendered. The heavy profile adds snapshots, history harvest, compilation, maintenance, and dataset work. Both share one lock, so an overlap exits cleanly instead of double-running.

That autonomy belongs to memory maintenance. ocbrain does not enqueue agent work, install skills, ship packages, send hosted requests, or delete private data on its own.

The dataset is the long game.

The same evidence compiles into three local streams: successful exchanges for SFT, corrections for DPO, and verified first-party writing for persona and voice. v0.4 draws a hard boundary between examples that belong in model weights and facts that should remain retrieval-only, grades a bounded candidate pool locally, finalizes only passing rows, and freezes the evaluation before training.

The second pilot improved and still failed the model-quality bar. It reused the first pilot's twenty prompts, references, rubric, held-out hashes, and blind randomization exactly. After eleven canonical first-party examples were added locally, the candidate improved from 2 preferences to 7; the reference still won 13.

The corpus, references, candidates, blind key, ratings, and adapter stay in the gitignored local dataset tree. There is no hosted dataset export path.

Current proof, dated instead of implied.

On 2026-07-10, the v0.4 source suite passed 520 tests. The byte-frozen 100-case retrieval benchmark reached 100/100 top-five relevance across ChatGPT, Codex, Claude Code, and OpenClaw, with every citation and privacy-scope check passing.

The local evaluator now calibrates against a separate set of eight human labels, not machine-authored expected answers. It passed 8/8 at the fixed 90% gate after the evaluator rubric—not the labels or threshold—was corrected.

The v0.3.3 ledger audit remains the latest full-chain snapshot: 306,974 events reproduced 101,641 beliefs exactly, with zero foreign-key violations and zero parentless search rows.

Those are snapshots, not timeless promises. The proof page separates code gates, runtime acceptance, and model quality so a green result in one lane cannot stand in for another.

Formerly OpenClawBrain.

The project began as OpenClawBrain and became the smaller ocbrain runtime: local first, source-backed, shared through MCP, and conservative about what earns memory. The domain stayed because the question stayed the same. How should an agent remember without turning arbitrary text into truth?
