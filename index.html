OpenClawBrain

A learned retrieval brain for OpenClaw.

Your OpenClaw agent should not relearn the same lesson twice.

OpenClawBrain is a fork of lossless-claw that adds a paper-faithful reinforcement learning layer for context routing. It combines lossless transcript memory with a knowledge graph whose routing policy learns from outcomes.

openclaw plugins install @jonathangu/openclawbrain

Two layers, one plugin

Lossless transcript memory

Every message persisted. Older chunks summarized into a DAG. Nothing lost. Inherited from lossless-claw.

Learned retrieval brain

A knowledge graph whose routing policy learns what context matters, from real usage signals.

How it learns

  • Policy-learning over edges — softmax routing with full-trajectory REINFORCE (not one-step)
  • Four label sources — human corrections > task outcomes > structural patterns > async teacher
  • Structural graph mutations — split, merge, prune, connect, inject — validated by replay gates
  • Immutable serving packs — learning never corrupts live serving

What the agent gets

Every turn, the brain assembles context in four sections:

  1. Correction cards (human-taught fixes, highest priority)
  2. Route-selected evidence (nodes the learned policy picked)
  3. Toolcards and playbooks (when/how to use specific tools)
  4. Transcript support (fresh tail + archival recall from the summary DAG)

Three tools

  • brain_teach — teach the brain a correction or pattern
  • brain_status — health card, pack version, learning stats
  • brain_trace — full decision trace for any query

The paper

The learning algorithm implements Lemma 6.1 from:

Jonathan Gu — Reinforcement Learning (Econometrics Field, July 2016)

Corrects Williams (1992) one-step REINFORCE to a full-trajectory update that assigns credit to every routing decision in the episode.
