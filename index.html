Architecture and package boundary

OpenClawBrain is a package surface, not a runtime replacement. Runtime orchestration stays in OpenClaw while learning contracts and artifacts stay in OpenClawBrain.

OpenClaw runtime owns

Session and channel orchestration, fail-open behavior, prompt assembly, diagnostics, and worker lifecycle.

OpenClawBrain packages own

Contracts, typed events, export normalization, provenance, pack format, activation helpers, compiler output, and learner policy updates.
