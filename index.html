OpenClawBrain for OpenClaw

Durable memory for an agent that should not relearn the same lesson twice.

OpenClaw gives your agent a runtime. OpenClawBrain gives it durable memory, learned retrieval, and decision traces you can inspect when you need to understand why a context choice won. The learned router now runs native V2 policy-gradient routing end to end — no legacy labels, no validator seam.

Install and verify

The public install lane is still plugin first, then CLI. Install the OpenClaw plugin, run the operator CLI against your OpenClaw home, restart the gateway, then verify with one status command.

Verified install lane: @openclawbrain/openclaw@0.4.0 plus @openclawbrain/cli@0.4.10 
openclaw plugins install @openclawbrain/openclaw@0.4.0
npx @openclawbrain/cli@0.4.10 install --openclaw-home ~/.openclaw
openclaw gateway restart
npx @openclawbrain/cli@0.4.10 status --openclaw-home ~/.openclaw --detailed

Verification command: npx @openclawbrain/cli@0.4.10 status --openclaw-home ~/.openclaw --detailed

Metadata note: the truthful native V2 metadata milestone was published through @openclawbrain/contracts@0.3.5 and consumed by @openclawbrain/cli@0.4.10, but the operator-facing install path remains plugin + CLI.

Operator note: the plugin install may warn that the manifest id is openclawbrain while the package or entry hint says openclaw. That warning is currently cosmetic; the flow above is the verified install path.

Proof: native V2 metadata, end to end

The learned router now reports exactly what it runs. The old seam — where native V2 routing still surfaced policy_gradient_v1-compatible labels at the package layer — is closed. Every metadata field the operator sees is truthful V2.

Remove cleanly

Removal mirrors installation. Use the CLI to uninstall from your OpenClaw home, remove the plugin by its manifest id, then restart the gateway. If you used npx above, there is no persistent CLI package to remove.

Full removal

Remove the installed brain data for ~/.openclaw, then remove the plugin record and files from OpenClaw. 

npx @openclawbrain/cli@0.4.10 uninstall --openclaw-home ~/.openclaw --purge-data
openclaw plugins uninstall openclawbrain
openclaw gateway restart

CLI package note

The recommended install path uses npx, so there is no global CLI to clean up afterward. Only remove the CLI package separately if you installed it globally on purpose. 

npm uninstall -g @openclawbrain/cli

OpenClaw uninstall uses the plugin id, so the command is openclaw plugins uninstall openclawbrain, not the npm package spec.

How it works

Remember what happened, retrieve the few pieces of memory that fit the next task, and learn from whether they helped.

1

Record durable memory

Sessions and explicit corrections stop being disposable chat and become searchable memory.

2

Find the right neighborhood

The system searches for the part of memory that actually fits the new task.

3

Surface only the best context

It injects the highest-value context instead of dumping every vaguely related note into the prompt.

4

Learn from outcomes

Helpful routes get stronger. Wasteful routes lose weight over time.

5

Show its work

Decision traces make the winning path explainable instead of magical.

What a real correction looks like

The loop is not abstract. A user corrects the agent in plain language, the correction becomes durable memory, and the next similar answer improves because the right context got retrieved in time.

User says

How should you answer me?

Agent says

I'll answer in long freeform paragraphs.

User says

Wrong — keep it concise, use bullets, and end with the next action.

Agent says

Got it. I'll keep answers concise, use bullets, and end with the next action.

User says

How should you answer me?

Agent says

Concisely, with bullets, and ending with the next action.

What got retrieved behind the scenes

The correction is stored as durable memory, retrieved before the next similar answer, and made visible in the trace so you can inspect why the new answer won.

  • the user's correction is committed as durable memory
  • on the next similar question, that correction is retrieved as priority context before the agent answers
  • the stale default answer loses because the retrieved correction outranks it
  • brain_trace shows the retrieved context, the fired correction, and the losing path
