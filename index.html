Install and verify

Install the plugin, run the CLI against your OpenClaw home, restart the gateway, and verify with one command.

Verified install lane: @openclawbrain/openclaw@0.4.0 plus @openclawbrain/cli@0.4.10 
openclaw plugins install @openclawbrain/openclaw@0.4.0
npx @openclawbrain/cli@0.4.10 install --openclaw-home ~/.openclaw
openclaw gateway restart
npx @openclawbrain/cli@0.4.10 status --openclaw-home ~/.openclaw --detailed

Verification command: npx @openclawbrain/cli@0.4.10 status --openclaw-home ~/.openclaw --detailed

Metadata note: native V2 metadata ships through @openclawbrain/contracts@0.3.5 and is consumed by @openclawbrain/cli@0.4.10. The install path stays plugin + CLI.

Operator note: the plugin install may warn that the manifest id is openclawbrain while the package says openclaw. This warning is cosmetic. The flow above is verified.

Proof: native V2 metadata, end to end

The router reports exactly what it runs. Every metadata field is truthful V2 — the old policy_gradient_v1 label seam is closed.

Learn proof: the published learn path produced and promoted pack-d099f991. 
STATUS ok
path.pg            = v2
path.method        = policy_gradient_v2
path.target        = trajectory_reconstruction
path.trajectories  = 344
traced.routes      = 344
traced.supervision = 32

344 traced routes, 32 supervision signals, 68 updates. Routing method: policy_gradient_v2 targeting trajectory reconstruction. This is the actual method the router learned with, not a carried-forward V1 label.

Remove cleanly

Removal mirrors installation. Uninstall the brain data, remove the plugin, and restart the gateway. If you used npx, there is no CLI package to clean up.

Full removal

Remove brain data from ~/.openclaw, then remove the plugin from OpenClaw. 

npx @openclawbrain/cli@0.4.10 uninstall --openclaw-home ~/.openclaw --purge-data
openclaw plugins uninstall openclawbrain
openclaw gateway restart

CLI package note

The recommended path uses npx, so there is no global CLI to remove. Only run this if you installed the CLI globally. 

npm uninstall -g @openclawbrain/cli

Uninstall uses the plugin id (openclaw plugins uninstall openclawbrain), not the npm package name.

How it works

Remember what happened. Retrieve the memory that fits the next task. Learn from whether it helped.

1

Record durable memory

Sessions and corrections become searchable memory instead of disposable chat.

2

Find the right neighborhood

Search memory for the slice that fits the new task.

3

Surface the best context

Inject the highest-value context — not every vaguely related note.

4

Learn from outcomes

Helpful routes gain weight. Wasteful routes fade.

5

Show its work

Decision traces make the winning path explainable.

What a real correction looks like

A user corrects the agent. The correction becomes durable memory. The next similar answer improves because the right context arrives in time.

User says

How should you answer me?

Agent says

I'll answer in long freeform paragraphs.

User says

Wrong — keep it concise, use bullets, and end with the next action.

Agent says

Got it. I'll keep answers concise, use bullets, and end with the next action.

User says

How should you answer me?

Agent says

Concisely, with bullets, and ending with the next action.

What got retrieved behind the scenes

The correction is stored as durable memory, retrieved before the next similar answer, and visible in the trace.

  • the correction becomes durable memory
  • on the next similar question, it is retrieved as priority context
  • the stale default answer loses because the correction outranks it
  • brain_trace shows the retrieved context, the fired correction, and the losing path
