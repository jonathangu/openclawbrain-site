Native OpenClaw plugin · v0.2.1

Latency-safe memory that learns from your corrections.

OpenClawBrain is a native OpenClaw plugin that stores redacted memory nodes in local SQLite, retrieves only the few memories that matter, and keeps full proof and audit surfaces. By default it stays local; if you explicitly configure a structured JSON model endpoint, it can distill corrections and learn routes automatically.

The problem

Agents forget the small things that matter: a correction about your build tool, a preference about how you want tests run, or a useful workflow from the last session. The usual fallback is stuffing more text into every prompt.

OpenClawBrain keeps the claim tighter. It stores redacted local memories, retrieves candidates from SQLite, and injects only a bounded slice when the turn looks like it needs help.

What v0.2 does

  • Corrections can stick automatically. With the LLM path enabled, user corrections and preferences become scoped memory nodes instead of handwritten notes.
  • Routing stays latency-safe. Most turns use cached policy plus local retrieval. Only ambiguous or high-signal turns may pay for one bounded planner call.
  • Prompts stay bounded. Candidate retrieval is local, and only 0–5 selected memories are injected.
  • Learning happens in the background. agent_end, tool observations, and async jobs update outcomes and route policy.
  • Everything is inspectable. Status, proof, graph, learn, and search routes show what the plugin is doing.

Install

Requires OpenClaw 2026.4.29 or later.

# Install the plugin openclaw plugins install clawhub:openclawbrain openclaw plugins enable openclawbrain # Enable the v0.2 path openclaw config set plugins.entries.openclawbrain.config.enabled true --strict-json openclaw config set plugins.entries.openclawbrain.config.mode '"balanced"' --strict-json openclaw config set plugins.entries.openclawbrain.hooks.allowPromptInjection true --strict-json openclaw config set plugins.entries.openclawbrain.hooks.allowConversationAccess true --strict-json openclaw config validate openclaw gateway restart

Optional: enable semantic distillation and route learning

The intended default is a local OpenAI-compatible endpoint. If you leave llm.enabled off, the plugin still exposes proof, graph, learn, and search surfaces, but automatic correction capture is not active.

openclaw config set plugins.entries.openclawbrain.config.llm '{ "enabled": true, "provider": "local", "baseUrl": "http://127.0.0.1:11434/v1", "plannerModel": "your-local-model", "feedbackModel": "your-local-model", "learningModel": "your-local-model" }' --strict-json openclaw config validate openclaw gateway restart

Legacy compatibility files

If you want the older file-backed modes (conservative or active), you can still keep local activation files. They are optional in v0.2.

mkdir -p ~/.openclawbrain/activation/main echo 'Use pnpm, not npm.' > ~/.openclawbrain/activation/main/corrections.md echo 'Working on OpenClawBrain plugin.' > ~/.openclawbrain/activation/main/context.md echo 'Run tests before claiming something works.' > ~/.openclawbrain/activation/main/tool-guidance.md

How it works

  1. Capture signals. Corrections, tool outcomes, and session-end feedback become candidates for memory or route learning.
  2. Retrieve locally. SQLite and cached policy find candidate memories for the current turn.
  3. Plan only when needed. Most turns avoid synchronous model work. Ambiguous or high-signal turns can use one bounded planner call.
  4. Inject a small slice. The prompt gets only a few selected memories, not your whole history.
  5. Learn in the background. Outcomes update freshness, usefulness, and route examples without blocking the main turn.

Modes

ModeBehavior
offAlways stay silent.
proof-onlyLog route and proof events, never inject.
conservativeLegacy file-backed compatibility mode with narrower injection behavior.
activeLegacy file-backed compatibility mode with broader injection behavior.
balancedThe main v0.2 path: local retrieval plus bounded planner use when justified.
aggressiveThe v0.2 path with a stronger bias toward retrieval and injection.

Verify

The plugin ships with inspectable HTTP surfaces so you can see current state instead of guessing.

openclaw plugins inspect openclawbrain --json curl http://127.0.0.1:18789/plugins/openclawbrain/status curl http://127.0.0.1:18789/plugins/openclawbrain/proof?limit=10 curl 'http://127.0.0.1:18789/plugins/openclawbrain/graph?limit=10' curl 'http://127.0.0.1:18789/plugins/openclawbrain/learn?limit=10' curl 'http://127.0.0.1:18789/plugins/openclawbrain/search?query=pnpm&limit=10'

Safety

  • Off by default. enabled starts as false.
  • Raw transcript storage is hard-disabled. rawTranscriptUpload and storeRawTranscript remain false.
  • Redaction happens before store and before LLM. Proof and storage stay privacy-first.
  • No network calls unless you explicitly configure an endpoint. The default intended path is local; remote models are opt-in.
  • Fail-closed on missing permissions. The v0.2 path requires prompt injection and conversation access permissions to be explicitly granted.
  • Plugin failure does not block the main agent. OpenClaw still runs if the plugin errors.

What it does not do

  • It is not a cloud memory service.
  • It is not a replacement for OpenClaw itself.
  • It does not dump your full history into every prompt.
  • It does not require synchronous model calls on most turns.
  • It does not auto-distill corrections unless you configure the LLM path.
