A learned memory layer for OpenClaw agents

Your agent gets a second brain — one that learns

Most AI agents search their notes by keyword and hope for the best. OpenClawBrain replaces that search bar with a trained guide that maps how your agent's knowledge connects, picks the right context for each question, and learns from corrections so the same mistake never happens twice.

How it works

OpenClaw handles conversations (the front desk). OpenClawBrain handles memory (the back office). If the brain is ever unavailable, the agent keeps working — no single point of failure.

Knowledge graph

Facts, rules, and corrections — connected as a map

Instead of a pile of text chunks, the brain organizes knowledge as a graph where nodes are facts and edges are real relationships. "This deploy policy depends on that rollback rule" is a connection, not two documents that happen to share words.

Learned routing

A trained function picks the right context

When a question arrives, the routing function walks the graph and selects the most useful nodes — not just the most similar-sounding text. Corrections and outcomes feed back into training, so the guide improves with every interaction.

Structural plasticity

The map reshapes as the world changes

Nodes connect, split, merge, and prune under policy control. Stale knowledge fades; active connections strengthen. Ablations show this is the single largest accuracy contributor across every benchmark family.

Quick start

Build the workspace and produce your first brain pack in four commands. 

corepack enable
pnpm install --frozen-lockfile
pnpm check
pnpm release:check          # proof tests + local package build
pnpm fresh-env:smoke        # verifies clean install end to end

Distributed as local tarballs for now (npm packages coming). Full walkthrough: setup guide. Integration with OpenClaw: integration guide.
