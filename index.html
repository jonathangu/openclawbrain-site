Persistent memory for OpenClaw

Your OpenClaw agent forgets between sessions. OpenClawBrain makes corrections stick.

OpenClawBrain gives OpenClaw persistent memory that learns from corrections, retrieves the right context for each task, and improves recurring workflows over time. It stays local, fails open behind OpenClaw, and gets more useful the more you use it.

What this changes on a real task

A simple example: an agent keeps missing the exact rollback rule during a deploy incident. The difference is not abstract. It shows up on the next similar task.

Before OpenClawBrain

The agent repeats the same retrieval mistake

After OpenClawBrain

What changes in practice

OpenClaw still runs the agent. OpenClawBrain changes what the agent remembers and what context it sees next time. In day-to-day use, that shows up in three ways.

Fewer repeated mistakes

Corrections become reusable memory

When you fix a mistake, OpenClawBrain records what went wrong and why. The next time a similar situation comes up, the agent can retrieve that correction instead of repeating the error.

Cleaner prompts

Retrieval gets narrower

A trained routing function picks the context that matters for each question instead of dumping in every vaguely related chunk. Prompts get smaller, and the signal-to-noise ratio improves.

Better recurring workflows

Repeated workflows improve

Outcomes and feedback feed back into training. Workflows your agent runs repeatedly get better over time as the routing function learns what context actually helps.

How it works

Three product ideas, in plain English.

Knowledge graph

Facts live as connections, not loose documents

Instead of a pile of text chunks, knowledge lives in a graph. "This deploy policy depends on that rollback rule" becomes a real edge, not two documents that just happen to share a word.

Learned routing

A trained router decides what to retrieve

When a question arrives, the router walks the graph and selects the most useful nodes. It learns from corrections and outcomes, so it gets better at picking the right context over time.

Structural plasticity

The memory graph reshapes as reality changes

Nodes connect, split, merge, and prune under policy control. Stale knowledge fades, active connections strengthen, and ablations show this is the largest single accuracy contributor across every benchmark family.

Current status

Alpha

OpenClawBrain is in early alpha. The benchmark story is ahead of the production-hardening story. The routing function, graph structure, and plasticity policy are working; the published install, attach, and restart path is verified on real OpenClaw profiles; and live-learning proof on attached installs remains separate until exported turns land and later served turns reflect promoted knowledge. Expect breaking changes before a stable release. Apache-2.0 licensed. Requires Node 20+.

Install for OpenClaw operators

If you want to try it now, use the published package and run openclawbrain install from a normal shell. The default flow provisions bge-large, then attaches the brain under a dedicated activation root for that OpenClaw profile.

The install, attach, and restart path is verified on real OpenClaw profiles. Use a dedicated activation root per profile by default. Skip embedder provision only with --skip-embedder-provision or OPENCLAWBRAIN_INSTALL_SKIP_EMBEDDER_PROVISION=1. The local Ollama teacher path remains optional.

1. Install the package

Start with the published npm package

The operator flow starts here, then runs openclawbrain install from a normal shell.

npm install -g @openclawbrain/openclaw@0.3.0
2. Attach one brain to one profile

Keep each OpenClaw profile isolated

Use a dedicated profile home and activation root for each profile you wire up.

openclawbrain install --openclaw-home "$PROFILE_HOME" --activation-root "$ACTIVATION_ROOT" --workspace-id "$PROFILE_NAME"
3. Restart and verify

Use the clean OpenClaw restart shape

Restart the gateway, then check status with the same env-cleared pattern.

env -i HOME="$HOME" PATH="$PATH" openclaw --profile <PROFILE_NAME> gateway restart
env -i HOME="$HOME" PATH="$PATH" openclaw --profile <PROFILE_NAME> gateway status
Rollback stays bounded

Detach without cross-profile spillover

One activation root per profile keeps rollback, disable, and cleanup limited to that profile.
