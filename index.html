OpenClawBrain for OpenClaw

Your agent forgets everything between sessions. This fixes that.

OpenClawBrain is a memory layer for AI agents. It keeps the corrections, preferences, and past outcomes that usually vanish when a session ends.

It runs locally. If the memory layer has nothing useful, your agent keeps running. Learning happens in the background.

Watch an agent learn in four turns

Most memory systems store facts. OpenClawBrain stores corrections — and brings them back before the agent makes the same mistake twice.

Turn 1

The first answer is wrong

User: How should you answer me?

Agent: I'll answer in long freeform paragraphs.

Turn 2

The user corrects it

User: Keep it concise. Use bullets. End with the next action.

Agent: Understood. I'll answer that way from now on.

Turn 3

The same task returns later

User: How should you answer me?

System: Matching correction found before the answer is written.

Turn 4

The behavior changes

Agent: Concisely, with bullets, ending with the next action.

Trace: The retrieved correction is visible with the path that won.

What changed: OpenClawBrain stored the correction after turn 2. On the next similar task, it retrieved that correction before the answer was generated.

Three things most memory systems do not do

OpenClawBrain is not just a place to dump notes. It is built to change future behavior, show its work, and stay out of the response path.

It learns which memories are useful

Many systems store memories and retrieve them by similarity. OpenClawBrain can promote corrections that helped and demote context that did not.

It shows its work

When an answer goes wrong, you can inspect the retrieved context and the route that won. The behavior is not a black box.

It learns in the background

Export, pack building, and promotion do not have to sit on the agent's response path. If memory has nothing useful, the agent still answers.

How it works

Four concrete steps. Store what mattered, retrieve it when needed, learn from outcomes, and keep the path inspectable.

Store

Save what changed the answer

Sessions, corrections, and outcomes go into the memory layer.

Retrieve

Bring back the right correction

Before a similar task, the agent can pull in the memory that matters.

Learn

Keep the paths that help

Helpful routes can gain weight over time instead of resetting every session.

Trace

Show what happened

The retrieved context and the winning route can stay visible for inspection.

Install or update, verify, and capture proof

Use the current public packages. If the host does not have OpenClawBrain yet, install it. If it already has OpenClawBrain, update it.

Install OpenClawBrain

openclaw plugins install @openclawbrain/openclaw
npx -y @openclawbrain/cli install --openclaw-home ~/.openclaw
openclaw gateway restart
npx -y @openclawbrain/cli status --openclaw-home ~/.openclaw --detailed
npx -y @openclawbrain/cli proof --openclaw-home ~/.openclaw --skip-install --skip-restart

Update OpenClawBrain

openclaw plugins update openclawbrain
npx -y @openclawbrain/cli install --openclaw-home ~/.openclaw
openclaw gateway restart
npx -y @openclawbrain/cli status --openclaw-home ~/.openclaw --detailed
npx -y @openclawbrain/cli proof --openclaw-home ~/.openclaw --skip-install --skip-restart

Use openclaw plugins install @openclawbrain/openclaw to install OpenClawBrain on a host that does not have it yet. Use openclaw plugins update openclawbrain to update an existing install. If you are updating an existing host, the plugin update is only step 1. You still need to rerun the CLI install command so the activation root and native package plugin wiring stay correct for that OpenClaw home. status --detailed is the quick verify surface. proof writes summary.md, steps.json, verdict.json, raw step logs, startup breadcrumbs, and proof pointers under one bundle directory. After install or update, the same surface is also available through the first-class openclawbrain proof --openclaw-home ~/.openclaw command. That proof lane freezes install/runtime/reporting truth; broader live-gain claims still need their own evidence. Need the expected fields? Compare your output with the proof page.
