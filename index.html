Stop reteaching your agent the same things.

OpenClawBrain is an OpenClaw plugin that remembers your corrections, carries useful context across sessions, and keeps prompts smaller. It runs locally. It does not upload anything.

The problem

You correct your OpenClaw agent. You tell it to use pnpm instead of npm. You fix a workflow. Next session, it forgets.

OpenClawBrain fixes that. It is a native OpenClaw plugin that hooks into the agent's pre-prompt build. It classifies each turn and, when it makes sense, injects bounded context from local files you control. On turns where it is not useful, it stays silent.

What it does

  • Corrections stick. You correct something once. OpenClawBrain brings it back when relevant.
  • Context carries forward. Notes, preferences, and working context are available across sessions.
  • Prompts stay small. Only useful context is injected, not your entire history.
  • Stays silent when not needed. On simple direct answers, it does nothing.
  • You can verify it. Status and proof routes show what it decided and why.

Install

Requires OpenClaw 2026.4.29 or later.

# Install the plugin openclaw plugins install clawhub:openclawbrain openclaw plugins enable openclawbrain # Configure it openclaw config set plugins.entries.openclawbrain.config.enabled true --strict-json openclaw config set plugins.entries.openclawbrain.config.mode '"conservative"' --strict-json openclaw config set plugins.entries.openclawbrain.hooks.allowPromptInjection true --strict-json openclaw config validate openclaw gateway restart # Verify it is running openclaw plugins inspect openclawbrain --json curl http://127.0.0.1:18789/plugins/openclawbrain/status

Create your context files

The plugin reads from local files. Create the ones that are useful to you:

mkdir -p ~/.openclawbrain/activation/main # Things you corrected the agent on echo 'Use pnpm, not npm.' > ~/.openclawbrain/activation/main/corrections.md # Ongoing context that helps across sessions echo 'Working on OpenClawBrain plugin.' > ~/.openclawbrain/activation/main/context.md # Hints for tool-heavy turns echo 'Run tests before claiming something works.' > ~/.openclawbrain/activation/main/tool-guidance.md

That is it. Use your agent normally. OpenClawBrain runs in the background.

How it works

OpenClawBrain hooks into the agent's pre-prompt build cycle. Every turn goes through four steps:

  1. Classify. The turn is classified: direct answer, correction follow-up, continuation, retrieval-heavy, tool-heavy, or stale memory conflict.
  2. Decide. Based on the mode, it decides whether to stay silent or inject context. Conservative mode only acts on corrections, continuations, and tool-heavy turns.
  3. Read and redact. If injecting, it reads the relevant activation files. Symlinks and oversized files are rejected. Content is redacted for emails, tokens, phone numbers, and URLs before injection.
  4. Audit. Every decision writes a local proof event. You can inspect what it did at any time.

Modes

ModeBehavior
offAlways stay silent.
proof-onlyClassify and log, never inject.
conservativeInject only on corrections, continuation, tool-heavy, and stale memory. Stay silent on direct answers.
activeInject on more slices. More aggressive.

Routes

EndpointDescription
/plugins/openclawbrain/statusPlugin status, config, last decision.
/plugins/openclawbrain/proof?limit=20Recent proof events.

Safety

  • Off by default. enabled is false until you turn it on.
  • Local only. No network calls. No data upload. No cloud service.
  • Redacted storage. Proof events store hashes of user text, never raw text. Every proof event asserts: rawTranscriptStored: false, rawUserTextStored: false, redactionApplied: true.
  • Fail-closed. If rawTranscriptUpload is set to true, the plugin shuts down entirely.
  • Not a single point of failure. If the plugin errors, the agent runs normally without it.

What it does not do

  • It is not a cloud memory service.
  • It is not a general intelligence layer.
  • It is not a replacement for OpenClaw's memory or context engine.
  • It does not upload raw transcripts or store raw user text.
  • It does not make model calls. Classification is heuristic.
