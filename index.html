Memory for OpenClaw

Your agent should not need the same correction twice.

OpenClawBrain helps OpenClaw agents remember useful corrections, preferences, and prior work — without stuffing your full history into every prompt.

It keeps memory local, searches it fast, brings back only a small useful slice, and gives you proof you can inspect. The result is simple: less repetition, fewer avoidable mistakes, and a system you can actually trust.

What it actually does

Remembers the useful stuff

When you teach your agent something practical — like which package manager to use, how you want tests run, or what matters in a repo — OpenClawBrain helps that lesson stick.

Keeps prompts small

It does not dump your whole history into every turn. It retrieves candidates locally and injects only a few memories when they are likely to help.

Shows its work

You can inspect status, run a health check, look at proof events, search memory, and inspect the graph instead of guessing what the plugin is doing.

Why this beats prompt bloat

The usual workaround for forgetful agents is to keep adding more text to the prompt. That is expensive, messy, and hard to trust.

OpenClawBrain takes a cleaner path:

  • store memory locally
  • search it quickly
  • bring back only what matters now
  • keep the whole thing inspectable

What is live today

OpenClawBrain is a real plugin today, not a concept page. The current release is v0.2.10. It supports a local memory graph, local search, health checks, proof routes, and a default local learning path aimed at Ollama out of the box.

If you deliberately turn automatic learning off, the plugin still loads and stays useful. You still get local memory surfaces and verification tools; you just do not get automatic distillation of new corrections.

Install in a minute

openclaw plugins install clawhub:openclawbrain
openclaw plugins enable openclawbrain

openclaw config set plugins.entries.openclawbrain.config.enabled true --strict-json
openclaw config set plugins.entries.openclawbrain.config.mode '"balanced"' --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowPromptContext true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowConversationAccess true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowToolObservation true --strict-json
openclaw config validate
openclaw gateway restart
