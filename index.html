Repeated workflows that improve over time

OpenClawBrain adds durable memory and learned routing to OpenClaw, so repeated workflows improve over time instead of reloading the same context every turn.

OpenClaw is an agentic runtime for code, research, and operations. OpenClawBrain is its memory and learning layer.

The problem

Standard retrieval does not learn from outcomes.

Embedding similarity selects chunks independently per query with no memory of what worked before. When the same type of task comes up again, the system starts from scratch. OpenClawBrain adds a learned graph router that reinforces routes that succeed and suppresses routes that fail.

Without OpenClawBrain With OpenClawBrain
Full-history prep before first useful routing Boot from existing files immediately; learn in the background
Same context reloaded every turn Compiled context improves with corrections and outcomes
No feedback loop from corrections Human, self, and scanner labels all feed the learner

How it works

Local traversal on the hot path. Learning in the background.

1. Boot

Start from files

Boot immediately from existing workspace files. No full pre-scan gate before the first useful routing.

2. Serve

Learned routing

The learned route function runs on the hot path with bounded compute. Context is compiled, not dumped.

3. Learn

Background updates

Async teacher updates, structural graph learning, and label harvesting stay off the hot path.

OpenClaw owns runtime orchestration and fail-open behavior. OpenClawBrain owns contracts, provenance, packs, compiler, and learner packages.
