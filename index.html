OpenClawBrain for OpenClaw

Durable memory for an agent that should not relearn the same lesson twice.

OpenClaw gives your agent a runtime. OpenClawBrain gives it durable memory, learned retrieval, and decision traces you can inspect when you need to understand why a context choice won.

One canonical operator lane

New installs and upgrades use the same path. Install the published front-door package, attach it to one OpenClaw home, restart the gateway, then verify with one status read.

Front door package: @openclawbrain/openclaw@0.3.5 
npm install -g @openclawbrain/openclaw@0.3.5
openclawbrain install --openclaw-home ~/.openclaw
openclaw gateway restart
openclawbrain status --openclaw-home ~/.openclaw --detailed

Verification command: openclawbrain status --openclaw-home ~/.openclaw --detailed

When the next release ships, stay in the same lane: change the version in the first command and rerun this sequence.

Remove safely

There are two remove stories. Use detach when you want the hook removed but the OpenClawBrain data kept. Use uninstall --purge-data when you want the hook and that install's data removed.

Detach and keep data

Safe default if you are backing out the integration and may reattach later. 

openclawbrain detach --openclaw-home ~/.openclaw
openclaw gateway restart

Uninstall and purge

Full removal for that install, including the data tied to its activation root. 

openclawbrain uninstall --openclaw-home ~/.openclaw --purge-data
openclaw gateway restart
npm uninstall -g @openclawbrain/openclaw

If you only want the hook gone and the data preserved, detach is the simpler keep-data path.

How it works

The public story stays simple because the runtime loop is simple to explain: remember what happened, retrieve the few pieces of memory that fit this task, and learn from whether they helped.

1

Record durable memory

Sessions and explicit corrections stop being disposable chat and become searchable memory.

2

Find the right neighborhood

The system searches for the part of memory that actually fits the new task.

3

Surface only the best context

It injects the highest-value context instead of dumping every vaguely related note into the prompt.

4

Learn from outcomes

Helpful routes get stronger. Wasteful routes lose weight over time.

5

Show its work

Decision traces make the winning path explainable instead of magical.

What a real correction looks like

The loop is not abstract. A user corrects the agent in plain language, the correction becomes durable memory, and the next similar answer improves because the right context got retrieved in time.

User says

How should you answer me?

Agent says

I'll answer in long freeform paragraphs.

User says

Wrong — keep it concise, use bullets, and end with the next action.

Agent says

Got it. I'll keep answers concise, use bullets, and end with the next action.

User says

How should you answer me?

Agent says

Concisely, with bullets, and ending with the next action.

What got retrieved behind the scenes

The correction is stored as durable memory, retrieved before the next similar answer, and made visible in the trace so you can inspect why the new answer won.

  • the user's correction is committed as durable memory
  • on the next similar question, that correction is retrieved as priority context before the agent answers
  • the stale default answer loses because the retrieved correction outranks it
  • brain_trace shows the retrieved context, the fired correction, and the losing path

Compatibility path

The older wrapper package still exists for legacy installs, but it is secondary now. Keep new installs on the canonical lane above. 

openclaw plugins install @jonathangu/openclawbrain@0.3.5
