What Is OpenClawBrain?

Help OpenClaw stop forgetting the useful stuff.

OpenClawBrain is a memory system for OpenClaw. It learns from surfaced history, corrections, and notes, then brings back a small useful slice when it matters instead of dumping raw history into every prompt.

The payoff is simple: less repetition, less prompt bloat, and a better shot that yesterday's fix is still available tomorrow.

In Plain English

It tries to remember lessons, not just logs.

1. Read what you already surfaced

Exports, notes, docs, and explicit corrections can become memory material.

2. Promote only what survives replay

Learning happens away from the live prompt, and promoted structure has to pass checks before it becomes active.

3. Serve a bounded slice

The runtime aims to return a few useful blocks, not your whole archive.

Why Should I Care?

It is for people who are tired of reteaching the same agent.

If you use OpenClaw enough, you eventually hit the same pain: the agent forgets a correction, misses a preference, or loses a pattern you already paid to discover. OpenClawBrain is meant to give those lessons a path back into the next run.

Less repetition

Stop restating the same fix

When you correct a workflow or preference, the point is to have that correction show up before the same mistake repeats.

More continuity

Carry forward what mattered

Useful past work can stay available across sessions instead of each run starting from zero.

Lean prompts

Keep the hot path small

A bounded memory slice is easier to use than stuffing full scrollback into every prompt and hoping the right line wins.

Why this matters: memory only helps if it changes the next answer without turning the prompt into luggage.

Why Is This Better Than Raw History Or Vanilla RAG?

The goal is not more memory. It is better memory selection.

Raw history can be useful as an archive. Naive memory can store notes. Vanilla RAG can retrieve similar text. None of those automatically solve the live-prompt problem: deciding what actually deserves to come back right now.

Raw history

Too much context, too little judgment

A giant transcript remembers everything and helps with very little unless you manually sift it first.

Naive memory / vanilla RAG

Retrieval is not the same as learning

Similarity search can fetch related text, but it does not automatically learn which past corrections actually improved behavior.

OpenClawBrain

Promote what proves useful

It harvests surfaced material, promotes structure only after replay checks, and serves a bounded slice through a learned route function.

Think notebook, not shoebox: the design goal is to surface a few lessons that change behavior, not spray yesterday's scrollback into every run.

Why Should I Trust It?

The public story stays tied to visible proof.

OpenClawBrain does not ask you to believe in invisible magic. The mechanism is documented, the source is public, and the healthiest reference host exposes named signals you can compare against.

Status STATUS ok

The exercised host was healthy enough to inspect and serve from.

Load proof loadProof=status_probe_ready

Runtime load is proven, not just inferred from files on disk.

Serving state serve state=serving_active_pack

The live path is serving a promoted pack instead of mutating truth on the hot path.

Learned route routeFn available=yes

The reference host already exposes a learned route function on the serve path.

Honest boundary: the live released packages remain @openclawbrain/openclaw@0.4.5 and @openclawbrain/cli@0.4.16. A fresh sterile published install still showed warning-level proof noise, so status --detailed is the sharper quick read and proof is supporting evidence, not the whole story.

What Does It Look Like In Practice?

From repeated correction to remembered correction.

This is the product promise in one plain-language example. The point is not photographic recall. The point is that a lesson that mattered before has a better path back into the next run.

Before

You keep repeating yourself

"Do not rewrite that migration again." "Keep the existing API shape." "Use the same fix as last time."

Without a useful memory layer, those instructions are easy to lose between sessions or bury in old scrollback.

After

The correction has a path back

OpenClawBrain can surface a promoted, bounded memory block before the same mistake happens again.

That is the difference between "I have seen this before" and "I learned from this before."

What Do I Do Next?

Try it on one pinned OpenClaw home.

The install lane is intentionally short and remains the canonical path. Use the same selected home on every openclawbrain command, restart once, then compare detailed status before you worry about proof noise.

Canonical install lane 
openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'

The restart command sits between install and verify, but it does not take the path flag itself.
