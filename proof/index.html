What proof covers

Use this page to confirm install, load, serving, and reporting truth on a healthy host, then reproduce that comparison on yours.

  • bundle verdict: success_and_proven means the captured bundle aligned across install, runtime, plugin load, breadcrumbs, runtime-load proof, and detailed status.
  • On the exercised March 27 host, STATUS ok lined up with loadProof=status_probe_ready, serve state=serving_active_pack, and routeFn available=yes. On fresh seed-state hosts, the load/serve/route lines can already be healthy while the headline still says STATUS warn.
  • proof writes summary.md, steps.json, verdict.json, raw step logs, startup breadcrumbs, and the runtime-load-proof snapshot into one durable bundle.
  • On published @openclawbrain/cli@0.4.16, status --detailed remains the stronger compare surface when fresh custom-home proof stays degraded on warning-level checks.

Pack ids and counters can differ by host state. Use the exercised-host surface as a reference, not as a requirement that every fresh host show STATUS ok.

Run the same lane on your host

Keep the same selected home on the OpenClawBrain commands before and after restart. Compare status first. Capture proof second.

Canonical commands

Install, restart, verify, prove

openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'
Why the path stays pinned

One selected home stays in scope

install, status, and proof all accept --openclaw-home. Keep them pinned to ~/.openclaw. openclaw gateway restart is the only step without that flag.

If you explicitly manage @openclawbrain/openclaw with the OpenClaw plugin manager, rerun openclawbrain install --openclaw-home '~/.openclaw' afterward so the hook and activation root converge again.

Published CLI 0.4.16 note: if the same host already shows runtime=proven, loadProof=status_probe_ready, serve state=serving_active_pack, and routeFn available=yes, treat detailed status as the stronger truth surface when proof stays degraded on warning-level checks. The old blocking verdict gap is fixed in the current npm release.
