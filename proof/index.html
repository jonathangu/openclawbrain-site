The lock-window result matters.

A previous implementation let retrieval-audit logging turn a successful read into database is locked. The current path separates the two outcomes. Search still returns scoped context, while the response says retrieval_use_status=database_busy and omits the feedback handle.

The Claude acceptance hit that exact condition. Claude used the successful read and did not retry merely to manufacture an audit id. That is a passed concurrency behavior. It is not a recorded Claude feedback result, and the documentation does not present it as one.

Six safeguards stand in front of injectable memory.

Injection scanning

Third-party evidence is scanned before it can reach the injectable class. A hit moves the row out of memory.

Provenance no-clobber

An automatic write cannot silently replace stronger first-party or human-authored provenance.

Secret scrubbing

Credential-like bodies are quarantined. Dataset and optional judge paths redact before they package text.

Privacy ratchet

Derived scope can only become more restrictive. Private evidence never widens into a public or global result.

Tripwire quarantine

Injection, secret residue, harmful-feedback spikes, hard corrections, belief thrash, and unverified risky content can remove a row immediately.

Hash-chained audit

Evidence, decisions, corrections, retrieval outcomes, and runs leave a history that can be inspected instead of overwritten.

The autonomy boundary is narrow on purpose.

ocbrain may harvest local history, compile knowledge, label outcomes, quarantine risky rows, promote memory, render bounded excerpts, and build local datasets. It does not get general authority from doing those jobs.

  • It does not enqueue or execute agent work.
  • It does not install skills or ship packages.
  • It does not send a hosted-teacher request merely because a preview passed.
  • It does not export the training corpus to a hosted endpoint.
  • It does not destructively delete the evidence ledger as ordinary maintenance.

The first fine-tune was a pipeline pass and a model-quality fail.

The blind evaluation existed before the training files. Twenty graded persona prompts and their references were held out, and their content hashes were removed from every training stream. The base model and trainer revision were pinned before the run.

The corrected 25-step local QLoRA run completed. Training loss ended at 2.218, and validation loss moved from 6.594 before training to 4.592. The adapter loaded and generated all twenty candidates.

The tuned candidate was preferred in 2 of 20 decided comparisons. Mean voice fidelity was 2.25 for the candidate and 4.60 for the reference. That fails the voice bar. It also tells us what to improve while the corpus is still cheap to change.

The useful conclusion is not that fine-tuning works. The useful conclusion is that the evaluation can reject a weak model before anyone starts believing its style is Jonathan's.

What could still be wrong.

  • Judge echo chamber. A grader can reward its own stylistic preferences unless it is calibrated against independent human decisions.
  • Long writer windows. Reads now degrade safely, but the architecture still has one SQLite writer and needs continued concurrency hardening.
  • Retrieval quality. A healthy MCP call does not prove the returned context was the best context. A labeled retrieval evaluation harness is still needed.
  • Single-machine fragility. The corpus has no automatic off-machine backup because a backup widens the privacy surface. That remains an operator decision.
  • Operator configuration. OpenClaw SecretRef migration and the enabled Policy plugin's policy are separate security choices. The ocbrain source cannot choose them honestly on the operator's behalf.
