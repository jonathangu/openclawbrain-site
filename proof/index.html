Safe & inspectable

Agents consult a brain. Humans decide what it remembers.

ocbrain's core idea is not "agents run themselves." It is that agents consult a source-backed brain, emit evidence, and let a controlled compiler decide what becomes memory. Everything risky is human-gated.

How the human gate works How to verify it

Read-first MCP surface

The tools an agent gets by default are all read-only. They retrieve and report; they never change durable memory.

brain.search

Find source-backed context relevant to the current turn. Read-only.

brain.digest

Report counts and state so memory quality is auditable. Read-only.

brain.get

Fetch a specific record and trace it back to its evidence. Read-only.

brain.feedback

Record whether served memory was helpful, used, ignored, irrelevant, or harmful. Read-only with respect to durable knowledge.

Write-capable tools are hidden unless the server is launched with --allow-writes:

# Default launch: read-only surface only
ocbrain serve

# Opt in to write-capable tools (brain.propose, brain.mark_stale)
ocbrain serve --allow-writes

Even with writes enabled, those tools stage candidates for review. They do not silently make knowledge current.

The human gate

Capability, prescriptive, and high-risk knowledge is never adopted automatically. It is staged as a candidate and requires a human to approve it through proposal-first review.

Nothing executable or loadable becomes current silently. A proposal sits as a candidate until a person reviews it and promotes it — or leaves it staged.
Capability → staged Prescriptive → staged High-risk → staged Human approves to promote

What ocbrain is NOT

ocbrain is a local SQLite evidence→knowledge ledger. It does not take actions on your behalf.

Not an autopilot

It does not act for the agent or the user.

Not a loop-runner

It does not execute loops.

Not a task scheduler

It does not enqueue or time work.

Not a skill installer

It does not install or activate skills.

Not a policy applier

It does not enforce or apply policies.

It can OBSERVE loops

It emits liveness tripwire evidence when a loop looks starved — without running or enqueueing anything.

Inspectable by design

Memory quality is auditable instead of vibes-based. You can always ask why ocbrain believes something.

  • Every current belief links back to its supporting evidence. There are no unsourced facts.
  • brain.digest exposes counts and state so the health of the ledger is visible at a glance.
  • Retrieval feedback is recorded — whether served memory was helpful, used, ignored, irrelevant, or harmful.
# See the state of memory: counts, states, evidence coverage
brain.digest

# Trace any current belief back to the evidence behind it
brain.get <id>

Maintenance keeps it honest

Upkeep is conservative and leaves a trail. It never hard-deletes.

prune

Marks stale knowledge that is unreferenced, or served but never useful. Useful feedback protects a row, and nothing is hard-deleted.

heal

Resolves conflicting current values by superseding the lower-confidence rows and writing correction evidence.

liveness-check

Writes tripwire evidence for silent loops — as an observer, never a scheduler.

Privacy

Scope only ever gets tighter, never looser.

  • Derived knowledge can only tighten scope to the most restrictive linked source.
  • Private evidence never gets promoted to a broader audience.

Because beliefs inherit the most restrictive scope of their sources, a private source can never leak into a wider audience through derivation.
