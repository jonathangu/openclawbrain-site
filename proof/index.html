OpenClawBrain verify

Check that it is running and doing its job.

After install, status --detailed is the fast check. It tells you whether the runtime is loaded, serving, and routing. The proof command captures a durable bundle when you need a record you can keep.

The public story is explicit about shipped versus target-state. Teacher v3 substrate is complete, and the runtime/proof adoption path is landed through replay-gated promotion and bounded canary discipline. Anything still future work stays labeled target-state on purpose.

When to use each command

Use status --detailed for the fast check. Use proof when you need a durable record.

  • status --detailed is the fast live check. It now exposes feedback and attrCover alongside the four core signals.
  • proof captures a durable bundle: summary.md, verdict.json, steps.json, logs, and runtime-load proof in one directory.
  • bundle verdict: success_and_proven means install, runtime, load proof, and detailed status all aligned in the captured bundle.
  • Pack ids and counters depend on host state. The four healthy signals matter more than exact numbers.
  • The public worked examples on How it works trace the same lifecycle: proposal, replay, bounded canary discipline, then promotion or shadow-only retention.
Current evidence: checked replay beats no_brain on real traces, and learned_route has real-trace wins. The replay set is still growing, so lower cost is plausible but direct spend savings are not yet proven.

Run the same check on your host

Keep the home directory the same, restart once, then compare status before you run proof.

Commands

Install, restart, status, proof

openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'
How to read the output

Status first, proof second

If status --detailed already shows the four healthy signals, the proof bundle should confirm and package them. A fresh host may still emit warnings in the proof output.
