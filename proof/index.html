What this page proves

The current public install lane can report concrete V2 metadata and capture one durable operator bundle. Verify it from the surfaced fields, not from fixed counts that only belong to one host.

  • path.pg = v2 reports the active policy-gradient generation.
  • path.method = policy_gradient_v2 reports the method by name.
  • path.target = trajectory_reconstruction reports the learning target.
  • traced.* fields show that traced routes, supervision, and updates exist.
  • proof captures one durable bundle with summary.md, steps.json, verdict.json, raw step logs, startup breadcrumbs, and runtime-load-proof snapshots.
Boundary: this is operator proof for install, runtime, and reporting truth. Your promoted pack id and traced counts will differ by host, and this page does not claim universal memory citations or live answer-quality gains by itself.

Example proof output

Run status and compare your output with the fields below. The exact pack id and counts vary by host and learning state. 

STATUS ok
path.pg            = v2
path.method        = policy_gradient_v2
path.target        = trajectory_reconstruction
path.trajectories  = <host-specific count>
traced.routes      = <host-specific count>
traced.supervision = <host-specific count>
traced.updates     = <host-specific count>

This output is useful because the method, target, pack, and traced-learning surfaces are real. The exact counts are not stable public constants.

Path fields

These lines identify the method and target that the router reports for this install.

Trace fields

These counts show that traced routes, supervision, and updates were recorded.

Reproduce it from the public install or update lane

Use the same setup path shown on the homepage. If the host does not have OpenClawBrain yet, install it. If it already has OpenClawBrain, update it first, then rerun the CLI install step, restart, run status for the quick check, and capture the proof bundle.

Install OpenClawBrain

openclaw plugins install @openclawbrain/openclaw
npx -y @openclawbrain/cli install --openclaw-home ~/.openclaw
openclaw gateway restart
npx -y @openclawbrain/cli status --openclaw-home ~/.openclaw --detailed
npx -y @openclawbrain/cli proof --openclaw-home ~/.openclaw --skip-install --skip-restart

Update OpenClawBrain

openclaw plugins update openclawbrain
npx -y @openclawbrain/cli install --openclaw-home ~/.openclaw
openclaw gateway restart
npx -y @openclawbrain/cli status --openclaw-home ~/.openclaw --detailed
npx -y @openclawbrain/cli proof --openclaw-home ~/.openclaw --skip-install --skip-restart

Use openclaw plugins install @openclawbrain/openclaw to install OpenClawBrain. Use openclaw plugins update openclawbrain to update an existing install. If you are updating an existing host, the plugin update is only step 1. You still need to rerun the CLI install command so the activation root and native package plugin wiring stay correct for that OpenClaw home. status --detailed is the fast compare surface. proof writes summary.md, steps.json, verdict.json, raw step logs, startup breadcrumbs, and runtime-load-proof snapshots into one bundle. After install or update, the same surface is also available through the first-class openclawbrain proof --openclaw-home ~/.openclaw command. If your OpenClaw home lives somewhere else, change the path.
