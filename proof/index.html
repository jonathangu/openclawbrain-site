Proof

Proof, not hand-waving.

Close it out means evidence, not vibes. OpenClawBrain is built so you can inspect what it is doing instead of trusting a vague claim that an agent “has memory now.”

These commands verify the accountable local system: plugin load, SQLite + FTS health, runtime status, proof events, graph state, local search, and learning surfaces.

Start here

openclaw plugins inspect openclawbrain --json
curl http://127.0.0.1:18789/plugins/openclawbrain/status
curl http://127.0.0.1:18789/plugins/openclawbrain/doctor

You want to see the plugin loaded, runtime routes and hooks present, and SQLite + FTS healthy.

Use inspect for runtime truth. A plugin list view can look like “zero plugins/routes/hooks” when it is only showing metadata snapshot fields.

End-to-end proof shape

The installed live loop was verified with a temporary memory: temporary correction → SQLite memory → FTS retrieval → XML prompt injection → cleanup.

These are the surfaces that let you tell the difference between “we say it works” and “you can inspect what it is doing right now.”

What the product claim is

OpenClawBrain is not claiming that every agent suddenly has perfect memory. The claim is tighter and more useful:

  • keep memory local
  • let the LLM decide semantic meaning without letting it write directly
  • retrieve likely candidates quickly with SQLite FTS
  • inject only a small useful XML block
  • show proof and health surfaces you can inspect

LLM decides semantic meaning. Code enforces trust boundaries. SQLite stores the graph and evidence.
