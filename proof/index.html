Benchmark + operator proof

Does it actually work?

by Jonathan Gu

We tested the full-brain mechanism — graph memory, learned routing, policy-gradient updates, and structural plasticity — against 8 baselines on three distinct long-lived memory tasks, each at 10-seed scale. The result: +9.7 to +26.9 pp over the best RAG baseline, a 29-1-0 head-to-head record across 30 seed-level comparisons, and 5x less context per query. Separately, the install/operator path is now verified on real OpenClaw profiles. This page keeps those proof layers separate and does not claim live learning before first export.

Cross-family headline

29-1-0 across three families

  • Relational drift (recorded H2H): 99.15% vs 89.44%, +9.71 pp, 10-0-0
  • Recurring workflows (simulation): 97.6% vs 70.6%, +26.9 pp, 10-0-0
  • Sparse feedback (simulation): 92.0% vs 67.0%, +24.9 pp, 9-1-0

10 seeds per family. 8 baselines each. 5x less context everywhere. Full artifacts in brain-ground-zero.

Operationally verified

Install, attach, restart

npm install -g @openclawbrain/openclaw@0.3.0 worked. The safest default is a dedicated activation root per profile. Attach succeeded on Tern and CormorantAI, and Tern came back alive on Telegram after restart.

This proves the public operator path, not live learning. Both attached brains are still awaiting_first_export.

The big picture: margin, context, and ablation

Two charts tell the cross-family story. The first shows how much full_brain wins by and how much less context it uses. The second shows where the advantage comes from — structural plasticity is the largest contributor in every family.

Family 1: Relational drift

Entity relationships change over time. The guide has to track who reports to whom, which teams own which services, and how these shift. This is the strongest evidence — recorded head-to-head with deterministic replay and hash verification.

Recorded H2H · 10 seeds · 800 queries/seed

99.15% vs 89.44%

  • Margin: +9.71 pp. Head-to-head: 10-0-0.
  • Context: 5x lower (800 vs 4,000 tokens per query).
  • Evaluation: deterministic fixture replay with SHA-256 verification. This is the strongest kind of evidence — fully reproducible, no simulation ambiguity.
  • 8 baselines tested: oracle, full_brain, vector_rag_rerank, vector_rag, heuristic_stateful, graph_route_pg, route_fn_only, and more.

Public bundle: proof-results/recorded_h2h_relational_drift_10seed.

Family 2: Recurring workflows

Repetitive tasks that the system should recognize and handle better over time — deploy checklists, review protocols, onboarding flows. This is where the full-brain advantage is largest: +26.9 pp.

Simulation · 10 seeds · 3,522 queries/seed

97.6% vs 70.6%

  • Margin: +26.9 pp. Head-to-head: 10-0-0.
  • Context: 5x lower.
  • Evaluation: simulated policy functions, deterministic given seed. This is the family with the largest query count (3,522 per seed) and the widest margin.

Public bundle: proof-results/recurring_workflows_10seed.

Family 3: Sparse feedback

Real teams don't correct every answer. This family tests the harder scenario: can the guide win when only ~19% of queries get explicit feedback? The advantage holds, but with an honest asterisk.

Simulation · 10 seeds · 1,800 queries/seed

92.0% vs 67.0%

  • Margin: +24.9 pp. Head-to-head: 9-1-0.
  • Context: 5x lower.
  • Variance note: The ±18.3% std on full_brain reflects high seed-level variance under ~19% explicit feedback coverage. Accuracy depends on which queries receive corrections. One seed loss prevents an unqualified "always wins" claim.

Public bundle: proof-results/sparse_feedback_10seed.

What this proves — and what it doesn't

Proven

The mechanism dominates RAG on three tasks

  • The full-brain mechanism — graph memory + learned routing + policy-gradient updates + structural plasticity — beats the best RAG baseline and all partial-brain ablations on three distinct long-lived memory tasks.
  • Each family runs at 10-seed scale with 8 baselines. Results are deterministic given a seed.
  • Structural plasticity is the single largest contributor in every family (ablation ladder holds without reversal).
  • Context efficiency: 5x lower across all families.
Not proven

Where the claims stop

  • Performance on memory_compaction (designed but not run).
  • Behavior at larger world sizes (current: 50 entities / 80 workflows).
  • End-to-end performance with live LLM routing (harness uses simulated policy functions).
  • Production latency or cost.
  • First-export learning proof on attached real profiles; both current brains are still awaiting_first_export.
  • Real-session head-to-head on product traces (spec defined, no scored results).
  • "Always wins" across all families (sparse feedback is 9-1-0).

Operator proof: what is real in the field

The benchmark package proves the mechanism. Separate operator checks now prove the install/attach/restart path on real OpenClaw profiles. Live learning still has not crossed the line yet.

1 · Benchmark proof

Mechanism evidence is strong

  • Recorded H2H exists for relational drift: deterministic replay, SHA-256 verified, 10-0-0.
  • Recurring workflows and sparse feedback extend coverage to two more long-lived-memory families.
  • The benchmark claim is superiority over RAG-style baselines on these tasks, not live-runtime learning proof.
2 · Install / attach / restart proof

Real OpenClaw profiles now attach cleanly

  • npm install -g @openclawbrain/openclaw@0.3.0 worked on a real operator path.
  • The safest default is one dedicated activation root per profile.
  • Attach succeeded on both Tern and CormorantAI.
  • Tern came back alive on Telegram after the verified restart path.
env -i HOME="$HOME" PATH="$PATH" openclaw --profile <PROFILE_NAME> gateway restart
3 · Live learning proof

Still pending first export

  • Install and attach prove the operator surface, not that export, learning, and promotion have already happened live.
  • Both attached brains are still awaiting_first_export.
  • The next honest proof boundary is first export landing, then later served turns reflecting promoted knowledge.

Why this matters

Trust the single turn, then trust the aggregate

  • You can inspect one turn end-to-end: what context was sent, what answer came back, what correction was captured.
  • The benchmark scores are meaningful because every row traces back to the same per-turn contract.
  • Corrections don't update the guide immediately — new knowledge takes effect on the next background cycle, like a librarian updating the card catalog overnight.
