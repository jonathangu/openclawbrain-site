Proof, packaged for operator review.

OpenClawBrain has mechanism proof and a recorded-session head-to-head benchmark for the product it claims to ship: OpenClaw-native memory, a locally served and bounded hot path, and a learned runtime route_fn improved off the hot path by Ultimate Policy Gradient. Claim labels

Use the same words everywhere

  • Proven now: the artifact already exists and can be rerun or inspected.
  • Needs head-to-head: the claim depends on fixed recorded sessions or shadow traffic.
  • Not claimed: there is no matching artifact yet, so the site should not imply certainty.

Current workflow-proof slice

This deterministic harness is narrow on purpose: it checks whether the exact required history or runbook nodes reach prompt context for 4 fixed workflow queries, then shows whether background teacher supervision improves the served local route_fn without putting the teacher on the hot path.

What this proves now

Useful from graph priors, better after background learning

  • Cold-start graph priors already recover both required nodes on 2 of the 4 fixed workflow queries.
  • per_query_matrix.csv and per_query_matrix.md flatten the slice into 16 deterministic scenario-by-mode rows with required node IDs, prompt-context node IDs, exact-target success, required coverage, and pointer turns.
  • The learned runtime route_fn improves later, off the hot path, and reaches exact-target success 4/4 by epoch 14 with required coverage 1.00.
  • pointer_chase averages 1.25 turns and still only reaches 1/4 exact-target success in this harness.

The matrix proves which memory nodes reached prompt context inside the deterministic sim. It does not prove downstream answer quality, recorded-session task success, scanner/harvester label quality on live histories, or production latency and cost on served OpenClaw traffic.

Recorded-session head-to-head

The first fair comparison layer: a frozen query set, fixed state, and one rubric applied to all modes. This is benchmark evidence from a recorded session, not a live production answer-quality claim.

Benchmark result

full_brain 0.975 vs best baseline 0.896

  • 800 queries, 41 training steps, 8 baselines compared.
  • full_brain accuracy: 0.975
  • vector_rag_rerank (best baseline): 0.89625
  • Delta: +7.9 percentage points

Public bundle: proof-results/recorded_h2h_relational_drift_001 in brain-ground-zero at commit 1c54302.

Proof boundary

What this does and does not prove

  • This is a recorded-session benchmark: frozen queries, fixed state, deterministic rubric.
  • It proves comparative accuracy on a controlled head-to-head with 8 baselines.
  • It does not prove live production answer quality, shadow-mode wins, or online rollout outcomes.
  • The next rungs on the evidence ladder are shadow-mode and narrow online rollout.

Evidence ladder

Each step answers a different question. The honest package is to show the current rung and the next missing rung, not to collapse them together.

1

Mechanism proof

Run deterministic simulations and ablations to show the learned router and confidence gate improve under controlled conditions.

This proves the mechanism. It does not prove product superiority.
2

Recorded-session head-to-head

800 queries, 41 steps, 8 baselines. full_brain accuracy 0.975 vs vector_rag_rerank 0.896 (+7.9 pp). Public bundle: proof-results/recorded_h2h_relational_drift_001.

This rung is reached. Evidence is benchmark/recorded-session, not live production.
3

Shadow mode

Mirror real OpenClaw traffic without serving it yet. Keep the same output budget and save disagreement traces, prompt size, latency, and cost logs.

This is where product evidence starts to harden.
4

Narrow online rollout

Enable the brain for a small traffic slice, keep fail-open behavior, and only publish claims backed by the saved artifacts from that slice.

This is product proof, not simulation proof.

Artifact checklist

If a claim is cited on the site, the matching artifact bundle should exist. This is the minimum useful retention policy.

Worked turn

One traceable operator turn

  • Stable turn_id and inbound summary
  • Exact activation + compile parameters
  • Returned bounded compiled context block
  • Response trace or answer text
  • Correction event and learner outcome output
Simulation run

Mechanism artifact bundle

  • Exact command and config
  • Commit SHA and seed
  • Output directory
  • Generated JSON, SVG, and PNG artifacts
  • Figure paths cited by the site
Head-to-head

Frozen recorded-session eval

  • Query or session-set path plus digest
  • State snapshot identifier
  • Rubric version and per-mode outputs
  • Per-query judgments
  • Sampled worked-turn bundles
Shadow or online

Same-turn comparison data

  • Traffic slice and dates
  • Brain-off and brain-on outputs
  • Prompt, latency, and cost logs
  • Saved disagreement traces
  • Rollout window and fallback policy
