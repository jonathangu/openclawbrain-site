Safe & inspectable

No approval queue. Automatic safeguards instead.

ocbrain v0.2 runs unattended, so trust can't depend on someone clicking "approve." Every write passes the same fixed invariants a careful reviewer would enforce — and only clean, corroborated results reach the memory an agent sees. The safeguards are mechanical, always-on, and leave a trail.

Why gates were replaced

v0.1 staged every risky belief as a candidate and waited for a human to promote it. That was safe, but it didn't scale: the brain grew only as fast as someone reviewed a queue, and a queue that gets busy gets skipped. v0.2 keeps the safety property — nothing dangerous becomes injectable silently — but enforces it with invariants that run on every write, on a 15-minute light and hourly heavy cadence, whether or not anyone is watching.

Gates don't scale. Invariants do. The goal was never "a human in the loop" for its own sake — it was "risky content can't reach the prompt unchecked." A fixed set of automatic checks delivers that continuously.

The safeguards

These run on the write path itself, so there is no unguarded way in.

Injection scanning

Harvested third-party text is scanned for prompt-injection patterns — role hijacks, "ignore previous," exfiltration links, hidden characters, encoded blobs — at the evidence layer, at write time, and again before anything is rendered into a prompt. External content is data, never instruction.

Provenance no-clobber

An automatic write can never overwrite a human-authored row. If it tries, the write is refused and logged as an audit breadcrumb. The first writer wins; later passes add evidence, not authorship.

Secret scrubbing

Bodies are scanned for credential and key patterns. A row that looks like it leaked a secret is quarantined out of memory, and secrets are stripped before any body is ever considered for the optional judge or written to a dataset.

Privacy ratchet

Scope only tightens. Derived knowledge inherits the most restrictive scope of its sources; private can never widen to a broader audience through derivation. This one-way rule is what makes unattended operation safe.

Tripwire quarantine

Injection, secret leaks, repeated harmful feedback, hard corrections, belief thrash, or unverified risky content each trip a wire that pulls the row out of memory instantly — a status change, not a queue.

Hash-chained audit

Decisions, quarantines, and refusals are appended to a tamper-evident event log. Nothing is hard-deleted; the record of what the brain did — and declined to do — stays intact.

The risky class still gets a higher bar

Removing the human queue did not flatten the risk model. Executable, prescriptive, and high-risk knowledge is held to a stricter promotion rule than an ordinary fact.

Nothing executable or prescriptive becomes injectable on vibes. That class must carry passed-verifier evidence or an explicit human approval signal before it can ever reach inject = 1 — the automatic replacement for the old human gate, aimed at exactly the dangerous class. If it serves without either, a tripwire quarantines it.

What ocbrain still is NOT

Autonomy is about how the brain learns, not about taking actions in the world. ocbrain compiles knowledge; it does not act for you.

Not an agent

It does not take actions on your behalf or in the world.

Not a loop-runner

It observes loops and emits liveness evidence; it never executes them.

Not a task scheduler

Its own 15-minute light and hourly heavy cadence maintains memory. It does not enqueue your work.

Not a skill installer

It compiles knowledge about skills; it does not install or activate them.

The dataset never leaves the machine

v0.2 compiles fine-tuning datasets from your own history. That makes the egress guarantee load-bearing, so it is enforced in code, not policy:

  • There is no hosted export path. Datasets are written to a local directory; the export target class is local-only by construction.
  • private-scope rows never export — regardless of any flag.
  • Every example is scrubbed for secret residue, high-entropy blobs, injected-memory leakage, and prompt-injection before it is stored.
  • The optional judge is the only outward call, and it is fenced: off unless a key is present, knowledge rows only, private dropped, bodies redacted, budget-capped, and every batch writes an egress-audit row.

Inspectable by design

Memory quality stays auditable, not vibes-based. You can always ask why ocbrain believes something — and see what it quarantined.

  • Every current belief links back to its supporting evidence. There are no unsourced facts.
  • Every label carries a confidence and the signals that produced it.
  • Every autopilot run is recorded — per-stage results, quarantines, and refusals — in a run ledger.
  • Retrieval feedback is recorded and drives the next promotion cycle.
# See the state of memory: counts, labels, quarantines, coverage
brain.digest

# Trace any current belief back to the evidence behind it
brain.get <id>
