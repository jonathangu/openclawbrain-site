TypeScript Packages

These are the building blocks of OpenClawBrain. Think of them like layers in a stack — from raw events at the bottom to the learning engine at the top. Currently distributed as local tarballs; npm publishing is the target.

  • @openclawbrain/contracts — shared type definitions (the common language all packages speak)
  • @openclawbrain/events — normalizes raw OpenClaw session events into a consistent shape
  • @openclawbrain/event-export — exports session data for the learning pipeline
  • @openclawbrain/workspace-metadata — workspace context and file metadata
  • @openclawbrain/provenance — audit trail: tracks where every piece of context came from
  • @openclawbrain/pack-format — the knowledge snapshot format (frozen, versioned bundles)
  • @openclawbrain/activation — promoting, switching, and rolling back knowledge snapshots
  • @openclawbrain/compiler — turns a snapshot into the context the model actually sees (same input, same output, every time)
  • @openclawbrain/learner — background learning engine that builds better snapshots over time
  • @openclawbrain/openclaw — the integration layer that plugs into the OpenClaw runtime
