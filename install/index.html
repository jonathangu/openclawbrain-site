Install

Install it, then prove it is live.

OpenClawBrain is meant to be easy to try. Install the plugin, enable it, restart the gateway, and check the runtime surfaces.

Requires OpenClaw 2026.5.2 or later. Current OpenClawBrain release: 0.2.16.

Quick install Verification commands

1) Install the plugin

openclaw plugins install clawhub:openclawbrain@0.2.16
openclaw plugins enable openclawbrain
openclaw gateway restart

Pinned installs are easier to debug. If you want the latest release instead, use clawhub:openclawbrain.

2) Basic config

balanced is the recommended default. It keeps normal turns light, but still lets OpenClawBrain capture feedback and inject useful memory when the turn calls for it.

openclaw config set plugins.entries.openclawbrain.config.enabled true --strict-json
openclaw config set plugins.entries.openclawbrain.config.mode '"balanced"' --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowPromptContext true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowConversationAccess true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowToolObservation true --strict-json
openclaw config validate
openclaw gateway restart

3) Local LLM learning path

The LLM path is for semantic updates: distilling feedback, judging route examples, and improving the graph. The model proposes structured JSON. Code decides what gets stored.

ollama list

openclaw config set plugins.entries.openclawbrain.config.llm '{
  "enabled": true,
  "baseUrl": "http://127.0.0.1:11434/v1",
  "routeModel": "qwen2.5:32b-instruct",
  "plannerModel": "qwen2.5:32b-instruct",
  "feedbackModel": "qwen2.5:32b-instruct",
  "learningModel": "qwen2.5:32b-instruct"
}' --strict-json
openclaw config validate
openclaw gateway restart

If Ollama is unavailable, the main agent should keep working. Known memories can still be searched locally; background learning gets quieter or retries later.

4) Optional: scope multiple agents

If you run multiple named agents, scope them explicitly. Each agent gets its own local activation root and memory graph.

openclaw config set plugins.entries.openclawbrain.config.scopes.agents '["main","pelican","bountiful"]' --strict-json
openclaw gateway restart

Use your real agent ids. Single-agent installs can skip this.

5) Check runtime truth

openclaw --version
openclaw plugins inspect openclawbrain --runtime
openclaw doctor

You want to see the plugin loaded, hooks and routes registered, and no “No active memory plugin” warning in doctor.

Use runtime inspect for the source of truth. Plain list views can show package metadata without proving the plugin is active.

6) Teach it one small thing

Give it a durable correction:

Use pnpm instead of npm in this repo.

Then inspect what happened through authenticated plugin routes:

/plugins/openclawbrain/proof?limit=10
/plugins/openclawbrain/search?query=pnpm&limit=10
/plugins/openclawbrain/graph?agentId=main&limit=10
/plugins/openclawbrain/explain-last

If it is working, the win should feel quiet: the next test/build turn should pick up the repo rule without you repeating it.

Fallback install

If ClawHub is rate-limited or package metadata is still settling, install the release archive directly.

curl -L -o /tmp/openclawbrain-0.2.16.tgz \
  https://github.com/jonathangu/openclawbrain/releases/download/v0.2.16/openclawbrain-0.2.16.tgz
openclaw plugins install /tmp/openclawbrain-0.2.16.tgz --force
openclaw plugins enable openclawbrain
openclaw gateway restart
