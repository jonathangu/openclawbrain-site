OpenClawBrain install

Attach OpenClawBrain to one pinned OpenClaw home.

This is the front door for a new host. It converges the hook, pins the activation root, and leaves you with the same simple operator lane every time: install, restart, status, proof.

The install lane also doubles as the safe repair lane for an existing host. That is deliberate: one command, one boundary, no second workflow to memorize.

What install does

A successful install gives you a pinned host boundary, an attached runtime, and a clean path to restart and verify.

Pin the home

Keep one selected OpenClaw home in scope

Use the same --openclaw-home value every time so the boundary stays readable.

Converge the hook

Repair the attachment path

The install command aligns the hook and the activation root for the selected host.

Prepare verification

Leave the host ready for restart and status

Install is not the last step. Restart and detailed status make the live runtime truth visible.

Canonical install lane

Run these in order on the same pinned home. The restart command is the only step that does not take --openclaw-home. 

openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'
Truth note: the published split packages are @openclawbrain/openclaw@0.4.6 and @openclawbrain/cli@0.4.18. The current install lane is: run openclawbrain install, restart the gateway, then trust status --detailed as the quick live check; use proof when you need a durable evidence bundle.
