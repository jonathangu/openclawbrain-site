Install OpenClawBrain

Fast honest path: install the plugin, turn it on, and verify that memory is really loaded before you trust it.

The goal is evidence, not vibes. You should be able to prove plugin load, SQLite + FTS health, local Ollama config, route events, search, graph, and prompt injection.

Requires OpenClaw 2026.5.2 or later.

1) Install the plugin

openclaw plugins install clawhub:openclawbrain@0.2.16
openclaw plugins enable openclawbrain

2) Turn it on

openclaw config set plugins.entries.openclawbrain.config.enabled true --strict-json
openclaw config set plugins.entries.openclawbrain.config.mode '"balanced"' --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowPromptContext true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowConversationAccess true --strict-json
openclaw config set plugins.entries.openclawbrain.config.hooks.allowToolObservation true --strict-json
openclaw config validate
openclaw gateway restart

balanced is the recommended default. It keeps the common path light and only does extra work when a turn looks like it needs help.

3) Optional: enable multiple agents/profiles

If your OpenClaw home has multiple named agents, list them explicitly. Each scoped agent gets its own activation root and local memory graph.

openclaw config set plugins.entries.openclawbrain.config.scopes.agents '["main","pelican","bountiful"]' --strict-json
openclaw gateway restart

Use your real agent ids. A single-agent install can skip this step.

4) Local Ollama learning path

The default route/planner/feedback/learning model is qwen2.5:32b-instruct. OpenClawBrain aims at the local OpenAI-compatible Ollama endpoint by default; set it explicitly if you want the config to be self-documenting.

ollama list

openclaw config set plugins.entries.openclawbrain.config.llm '{
  "enabled": true,
  "baseUrl": "http://127.0.0.1:11434/v1",
  "routeModel": "qwen2.5:32b-instruct",
  "plannerModel": "qwen2.5:32b-instruct",
  "feedbackModel": "qwen2.5:32b-instruct",
  "learningModel": "qwen2.5:32b-instruct"
}' --strict-json
openclaw config validate
openclaw gateway restart

Ollama proposes; code disposes. If local model calls are unavailable, the main agent keeps working and memory gets quieter rather than becoming a single point of failure.

5) Check that it is really live

openclaw --version
openclaw plugins inspect openclawbrain --runtime
openclaw doctor
  1. Make sure the plugin status is loaded.
  2. Make sure runtime routes, hooks, and service are present.
  3. Make sure OpenClaw doctor no longer reports “No active memory plugin”.

Use openclaw plugins inspect openclawbrain --runtime for runtime truth. HTTP plugin routes use gateway auth on normal installs, so use your authenticated dashboard/client or pass your gateway auth header when using curl.

6) Teach it something small

For example:

Use pnpm instead of npm in this repo.

Then inspect what happened:

/plugins/openclawbrain/proof?limit=10
/plugins/openclawbrain/search?query=pnpm&limit=10
/plugins/openclawbrain/graph?agentId=main&limit=10

If the system is doing its job, the win should feel almost invisible: you stop repeating yourself, and the agent quietly does the right thing.
