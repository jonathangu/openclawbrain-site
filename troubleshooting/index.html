OpenClawBrain troubleshooting

Something looks wrong. Keep the scope tight.

The fix is almost always the same: keep the same home directory, rerun install, restart the gateway, and compare status --detailed. Only then decide whether proof is actually the problem.

Daemon-vs-hook skew and half-converged states surface as explicit blocks instead of silent passes. A fresh host can still show STATUS warn even when the runtime signals are healthy. That is a noisy proof surface, not a broken runtime.

Fast triage

Start with the signal that failed. Do not change home directories in the middle of the diagnosis.

Load not proven

loadProof=status_probe_ready is missing

Rerun openclawbrain install --openclaw-home '~/.openclaw', restart the gateway, and check status --detailed again.

Status warns

STATUS warn on a fresh host

If detailed status shows the healthy runtime signals, the host is fine. Proof warnings on a fresh host are expected.

Not serving

serve state=serving_active_pack is missing

Not always an install failure. A fresh or lightly used host may not have promoted packs yet.

No route function

routeFn available=yes is missing

Check whether the runtime is serving a promoted pack. Compare against the verify page reference instead of assuming every host should match the exercised one.

Daemon-vs-hook skew

Daemon running but hook not wired, or vice versa

status --detailed flags this explicitly. A half-converged state (one side active, the other missing) blocks with a clear message. Rerun the install and restart to bring both sides into alignment.

The healthy reference: the exercised host shows STATUS ok with load proof, serving state, learned route, and daemon-vs-hook alignment. If yours does not, the next step is usually reinstall plus restart.

Recovery commands

When in doubt, rerun the four commands in order. The restart is the only step without --openclaw-home. 

openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'
  • 1. Install

    Repair the install

    Rerun install to repair the plugin and memory layer.

  • 2. Restart

    Reload the runtime

    Restart makes the gateway pick up the updated state.

  • 3. Verify

    Check status before proof

    Detailed status is the faster check when proof output is noisy.

Two mistakes to avoid

Wrong directory

Changing the home directory mid-debug

If install, status, and proof do not point at the same home directory, you are comparing different installs.

Manual edits

Editing plugin files directly

If you changed plugin files by hand, rerun the install commands so the plugin and memory layer get back in sync.
