How it works · current source

A local knowledge brain that improves itself while you work.

ocbrain reads the history your AI agents already produce, compiles it into source-backed knowledge, decides what worked and what did not, promotes only the trustworthy parts into an injectable memory, and builds a local fine-tuning dataset from your own decisions. It runs on a 15-minute light and hourly heavy cadence on your machine.

From a review queue to a self-maintaining evidence loop

The first version was a careful librarian: it filed evidence and compiled knowledge, but a human had to approve anything durable. That gate was the bottleneck — the brain could only grow as fast as someone clicked "approve."

The current source replaces that general queue with automatic safeguards. Every write passes the same invariants a careful reviewer would enforce — provenance no-clobber, injection scanning, secret scrubbing, a privacy ratchet, tripwire quarantine, and a hash-chained audit trail — and only clean, corroborated results reach the memory an agent sees.

The design bet: a fixed set of mechanical safeguards, applied to every write, is safer and faster than a human queue that gets skipped when it's busy. Gates don't scale. Invariants do.
Human approval queue → automatic safeguards Manual promotion → scored auto-promotion Read-mostly server → autonomous 15-min / hourly loop

The pipeline, end to end

One direction, five movements. Raw agent history becomes trustworthy, injectable memory — and a byproduct dataset — without any step being skipped or any history being overwritten.

Datasets are drawn with a dashed edge because they are a read-only byproduct: they are compiled from the same evidence but never feed back into what an agent sees at runtime.

1 · Evidence — what actually happened

Everything starts as evidence: immutable, append-only, hash-pinned records of what happened. Agent transcripts, task closeouts, correction events, verifier outputs, test and deploy results, loop iterations, liveness tripwires. Evidence is never edited in place — a correction is a new record, and the original stays exactly as written.

Before any harvested third-party content can influence memory, it is injection-scanned. External text — a web page, another tool's output, a pasted log — is treated as data, never as instructions. Anything that looks like a prompt-injection attempt (role hijacks, "ignore previous," exfiltration links, hidden characters, encoded blobs) is flagged at the evidence layer so it can never quietly become a belief.

2 · Knowledge — the current belief

A compiler turns evidence into knowledge: the current belief, not raw memory. Every piece of knowledge links back to the evidence that justifies it — a belief with no evidence is not a valid shape. Knowledge comes in three types, and it supersedes rather than overwrites, so the trail of what was once believed stays intact.

value

Facts and metrics — the concrete, queryable truths the brain currently holds.

doc

Readable wiki and procedure pages — the prose an agent or human reads.

capability

Executable or loadable skills and procedures the brain can hand an agent.

A dedicated safeguard protects this step from the brain editing over its own — or your own — work. The no-clobber rule: an automatic write can never silently overwrite a human-authored row. If it tries, the write is refused and the attempt is recorded as an audit breadcrumb. The first writer of a fact wins; later automatic passes add evidence, they don't rewrite authorship.

3 · Labels — good, bad, or neutral

This is the heart of the system. The brain reads its own outcomes and decides which knowledge earned trust and which misled it, using a 19-signal taxonomy. Signals are mined from the history that already exists — no extra instrumentation — and each is a small, dated, weighted vote attached to a piece of knowledge.

Human feedback

Corrections, thanks, approvals, reverts. When you correct an agent, that's a strong negative signal on whatever it just relied on; a "perfect, ship it" is a positive one.

Verification

Tests passing or failing, deploys succeeding or rolling back, verifier evidence. Verified beats claimed.

Retrieval outcomes

Whether a memory that was served turned out helpful, used, ignored, irrelevant, or actively harmful.

Operational & maintenance

Commitment outcomes, scheduled-job results, error-recovery arcs, and conflict-heal supersessions the brain resolved on its own.

Signals are folded into a label with time decay — recent evidence outweighs stale evidence — and a mass threshold, so a single weak vote can't flip a belief. A hard negative (an explicit "this is wrong") always wins outright. The result is written back as good, bad, or neutral with a confidence score.

For the genuinely ambiguous middle, an optional, budget-capped judge can weigh in on targeted knowledge rows. It is off unless configured, never grades the dataset, drops private content before packaging, secret-scrubs every body, and writes an egress-audit row for each batch. Its vote is one signal among many and cannot override a hard human correction.

4 · Memory — only the trustworthy parts

Memory is a read-only view over knowledge that has earned its place. Promotion into it is not automatic-by-default — it is scored. A row only becomes injectable when every one of these holds:

  • Current and not quarantined. A tripwire can pull any row out instantly.
  • Labeled good, with confidence above threshold. Neutral and bad rows do not inject.
  • Injection-scan clean. The body and every linked source are re-scanned before inject = 1.
  • Risky classes carry verification. Executable or prescriptive knowledge additionally needs passed-verifier evidence or an explicit human approval signal — the automatic replacement for the old human gate on exactly the dangerous class.
-- memory is a projection, not a store
CREATE VIEW memory AS
  SELECT * FROM knowledge
  WHERE status = 'current'
    AND inject = 1
    AND quarantine_reason IS NULL;

Promotion is ranked by a score blending confidence, freshness, and how useful a memory has actually been when served, and the injectable set is budget-bounded — a fixed cap on rows and characters, so memory stays sharp instead of bloating. A label that flips to bad, a confidence drop, or a quarantine demotes immediately. Human-pinned rows are never auto-demoted by score.

Tripwire quarantine — the safety net

Quarantine is the mechanical stand-in for a human saying "wait, not that." It is a status change, not a queue: a flagged row leaves the memory view, leaves retrieval, and leaves promotion the instant a tripwire fires — no one has to be watching.

injection_suspected

A linked third-party source, or the body itself, trips the injection scanner.

secret_leak

The value or title looks like it contains a credential or key.

bad_feedback_spike

Repeated harmful or failed retrieval outcomes in a short window.

hard_correction

An explicit, durable "this is wrong" targeting the row.

contradiction_thrash

The belief keeps flipping — superseded and re-asserted over and over.

prescriptive_unverified

Risky knowledge serving with no passed verifier and no approval signal.

Quarantine writes its own tripwire evidence and an audit event, and there is exactly one path back out — an explicit release. The append-only audit chain stays intact throughout.

Privacy only tightens

Every record carries a scope, from least to most restrictive:

private workspace project public

Derived knowledge inherits the most restrictive scope of everything it links. A public doc that cites a private source becomes private. Scope is a one-way ratchet — it can be pulled tighter by what a belief depends on, but private contents can never leak up into a broader audience through derivation. This is the invariant that lets everything else run automatically.

5 · Dataset factory — your decisions become training data

The same evidence that feeds memory also compiles into fine-tuning datasets. This is the long game: over time, the brain accumulates the material for a model trained on how you actually work. Three streams, each carrying full provenance:

SFT

Supervised chat examples — context plus a good final answer — mined from exchanges that ended well: an affirmation, a clean multi-step task success, a recovered error.

DPO

Preference pairs mined from corrections. When a first attempt was corrected and a later attempt was accepted, that becomes a rejected/chosen pair. Your corrections are the training signal.

Persona

Voice examples where your own verified messages and authored commits are the assistant target — so a fine-tuned model can learn your style, not a generic one.

Every example carries its source evidence ids, a privacy scope, a quality label, and a confidence. A quality scrub runs on each one — secret residue, high-entropy blobs, length bounds, near-duplicates, refusal-only answers, injected-memory leakage, and prompt-injection inside the target are all excluded before storage. Export is byte-deterministic with a signed manifest.

The hard guarantee: the dataset is written to a local directory and there is no hosted export path in the code. private-scope rows never export regardless of flags. The dataset never leaves the machine.

Grade locally, and build the evaluation before training

Mining examples is not the same as earning a training corpus. dataset-grade applies a different local rubric to SFT, DPO, and persona rows, rejects non-loopback endpoints before reading an example, and records the model, prompt version, progress, and normalized grade.

The pilot reserves at least twenty graded persona prompts, writes their references and voice rubric, and removes their content hashes from every training stream. Only after that frozen evaluation exists does it write the training pack. Candidate responses are randomized against the references, the rater never reads the A/B key, and scoring is the one step that unblinds the result.

A later pilot can reuse those files byte-for-byte while changing only the training corpus. Before its local judge opens a blind pair, the judge must pass a separate calibration set against a complete human-label file with named provenance. Machine-authored winners embedded in cases are ignored, and calibration-only mode stops before blind material is read. That keeps a better training run from quietly moving either the test or the evaluator.

That order matters. It keeps a training run from quietly changing the test it is supposed to pass.

The autopilot loop

All of this runs as one idempotent, single-instance state machine on a 15-minute light and hourly heavy cadence. The dispatch table has fourteen work stages, with lock and finalize wrapped around it by the runner. Any stage can be killed and safely re-run; watermarks, stable ids, and uniqueness constraints mean nothing is double-counted and no committed progress is lost.

  1. Lock

    Single-instance file lock. A slow run never collides with the next tick — the second invocation just exits.

  2. Snapshot

    A daily rotated copy of the database before anything is touched. Snapshot or migrate failing aborts the whole run.

  3. Migrate

    Additive-only schema migration — new columns and tables, never a destructive rebuild.

  4. Harvest

    Fingerprint-gated import of new history into evidence rows.

  5. Injection-scan

    Scan new third-party evidence before it can reach anything injectable.

  6. Review

    Post-turn review of settled sessions, committing each complete session before the lazy iterator parses the next transcript.

  7. Compile

    Turn undecided proposals into knowledge, rebuilding the projection once at the end.

  8. Auto-label

    Mine signals, release the writer before expensive attribution and hosted judging, fold into labels, then fold again.

  9. Embed

    Commit the egress audit, release SQLite for network I/O, then store each completed vector batch.

  10. Tripwires

    Run the tripwire registry and auto-quarantine anything that fires.

  11. Promote

    Re-score, promote the trustworthy rows into memory, demote what slipped, enforce the budget.

  12. Excerpt-render

    Render the promoted rows into the excerpt form the injectable memory actually serves.

  13. Maintain

    Prune stale knowledge, heal conflicts, and independently check loop and stall-watcher heartbeats.

  14. Dataset-mine

    Mine SFT, DPO, and persona examples from newly settled history in bounded, measured writer batches.

  15. Dataset-export

    Deterministically write the JSONL corpora and manifest — skipping the write if nothing changed.

  16. Finalize

    Record the run in a ledger — per-stage results or errors — and release the lock.

Each independent stage runs in its own try/except: a failure is recorded and downgrades the run to partial, but later independent stages still run. Only the two foundational stages — snapshot and migrate — abort the whole run, because everything after assumes a snapshotted, migrated database.

The grit contract has an independent witness

Agent instructions forbid ending a work turn with passive waiting: finish with environment-verified completion or an explicit blocked report. The separate stallcheck job then inspects workflow ledgers, runner state, ingress timeouts, and its own heartbeat. Stable fingerprints deduplicate findings; delivery state survives restarts; failed pages remain retryable; findings too old to page retire once instead of staying new forever.

Stallcheck does not claim or resume work. It records and pages. Autopilot maintenance watches stallcheck's heartbeat; autopilot commits a visible running row and moving deadline after every stage; stallcheck watches that deadline. Neither process is its own only witness. This is durable across process restarts on the same machine, but not across disk loss or removal of both launchd jobs.

What stays true by construction

Evidence before belief

No durable knowledge is invented without a source that traces back to something that happened.

Verified is not claimed

Passing tests and verifier evidence outrank confident-sounding text.

Memory is a view

There is no second store to keep in sync — update the knowledge, the view follows.

Supersede, never overwrite

History is appended, not rewritten. You can always ask what was once believed.

External content is data

Harvested third-party text is scanned and quarantined, never trusted as instruction.

Local by construction

The datasets have no hosted export path. Private scope never leaves. The machine is the boundary.
