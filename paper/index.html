Current benchmark evidence

RAG retrieves chunks by similarity. OpenClawBrain replaces that with a learned router over a knowledge graph — one that follows structural relationships, learns from corrections, and uses a fraction of the context. The benchmarks below show the gap on two frozen test suites. The paper tracks these published artifacts while keeping benchmark claims cleanly separate from shadow-mode and live-traffic claims.

Recorded-session head-to-head

full_brain 0.975 vs vector_rag_rerank 0.896

Artifact: recorded_h2h_relational_drift_001. Across 800 frozen queries against 8 baselines, the learned router answered correctly about 8 percentage points more often than the best RAG system — roughly 1 in 13 queries where RAG fails, the learned router gets right.

Sparse-feedback 10-seed

full_brain 91.96% vs best RAG 67.05%

With minimal correction signals, the router still outperforms the best RAG baseline by nearly 25 percentage points — while sending roughly 5x less context to the model per query. It won 9 out of 10 independent seed runs. RAG has no feedback loop, so the gap widens as learning compounds.

What the paper covers

The paper is the full technical write-up. These are its four main claims — each backed by mechanism and benchmark evidence, not yet by live-traffic proof.

System split

Fast serving stays local; learning happens in the background

When a user asks a question, OpenClaw serves it immediately using the current learned route_fn. Afterward — off the hot path — the system replays the turn, labels outcomes, and updates routing weights so future answers improve. Learning never adds latency.

Route signals

Combining graph structure with query-specific relevance

The router blends two signals: the graph's durable structure (graph_prior — what it already knows about how facts relate) and a per-query relevance score (QTsim — what fits this specific question). A learned confidence gate shifts weight between them: when the query signal is clear, lean on it; when it is ambiguous, fall back to structure.

Learning rule

One learning loop with a clear priority order

All learning — whether from human corrections, self-evaluation, harvested signals, or a teacher model — feeds into a single update loop. Human corrections always take priority, followed by the system's own outcomes, then everything else.

Maintenance path

The knowledge graph maintains itself

Instead of manual curation, the system treats graph upkeep — connecting, splitting, and merging nodes — as a learning problem. Conservative guardrails keep changes safe, and evidence gates each action class.
