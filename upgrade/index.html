OpenClawBrain upgrade / repair

Same commands, existing install.

Upgrade is not a separate workflow. When a host needs a package refresh or a repair, rerun the same four commands on the same home directory, then restart and verify.

One path for new installs. The same path for upgrades.

See the commands New install Troubleshoot

What upgrade means here

The goal is to get an existing install back to the same clean state as a fresh one.

Refresh packages

Pick up the current release

The current release is OpenClawBrain 0.4.26.

Repair drift

Bring the install back into alignment

The install command repairs the plugin and memory layer without making you learn new commands.

Verify again

Restart, then check status

After the rerun, status --detailed is the fastest way to confirm the runtime is healthy.

Upgrade commands

The same commands you would use for a new install. The difference is why you are running them, not what you type. 

openclawbrain install --openclaw-home '~/.openclaw'
openclaw gateway restart
openclawbrain status --openclaw-home '~/.openclaw' --detailed
openclawbrain proof --openclaw-home '~/.openclaw'
Note: a fresh install may still show STATUS warn in the proof output even when the runtime signals are healthy. That is expected. Use status --detailed as the primary health check, not proof alone.

When to use this page

Outdated packages

Your install is older than the current release

Rerun the install so the host picks up the current runtime and CLI packages.

Plugin drift

The plugin or memory layer needs repair

Use this page when the install is present but no longer working cleanly.

Post-change verify

You want to confirm the runtime still works

Restart and detailed status tell you whether the runtime is healthy before you need the full proof bundle.

Go deeper

If the install still looks wrong after the rerun, go to troubleshooting or compare against the healthy reference.
