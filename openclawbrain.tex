\title{OpenClawBrain: A Learning Memory System That Beats RAG on Long-Lived Agent Tasks}
\author{Jonathan Gu}

\begin{abstract}
Long-lived AI agents need memory that changes shape as the world changes.
Retrieval-augmented generation (RAG) treats memory as a static index:
embed everything, retrieve the top-$k$ chunks, and hope the answer is in them.
That works for single-turn lookups, but it breaks down when relationships drift,
workflows recur with variation, and feedback is sparse.

OpenClawBrain is an artifact-side learning system that replaces the static index
with a graph memory that rewires itself.
It combines a learned routing function, policy-gradient updates, and structural
plasticity---the ability to connect, split, merge, and prune nodes under policy
control---so that the memory topology tracks how the world actually evolves.

We evaluate the full mechanism against the best RAG baseline
(\texttt{vector\_rag\_rerank}) and seven other baselines across three benchmark
families, each at 10-seed scale (30 seed-level comparisons total).
Results:
\begin{itemize}
  \item \textbf{Accuracy:} +9.7\,pp to +26.9\,pp over best RAG across families
        (relational drift 99.15\% vs 89.44\%;
         recurring workflows 97.6\% vs 70.6\%;
         sparse feedback 92.0\% vs 67.0\%).
  \item \textbf{Head-to-head:} 29-1-0 across 30 seed-level comparisons.
  \item \textbf{Context cost:} 5$\times$ lower context per query than the best RAG baseline.
\end{itemize}

Ablations confirm that structural plasticity is the single largest contributor
in every family.
OpenClawBrain ships as an artifact-side package behind the OpenClaw runtime
boundary; it owns memory and learning while OpenClaw owns orchestration and
fail-open control.
\end{abstract}

\section{System Overview}
\label{sec:overview}

OpenClawBrain exists to solve a specific problem: agents that live for weeks or
months need memory that keeps up with a changing world, and today's dominant
approach---retrieve-then-read over a vector index---does not keep up.
The core result is that a graph memory with learned routing and structural
plasticity outperforms the best RAG baseline by +9.7 to +26.9 percentage points
across three distinct long-lived memory tasks, while using 5$\times$ less
context per query.

The system is organized around a clean boundary.
\textbf{OpenClawBrain} (artifact side) owns memory structure, routing, and
learning.
\textbf{OpenClaw} (runtime side) owns orchestration, tool dispatch, and
fail-open control.
This boundary is load-bearing: the artifact side can be tested, versioned,
and benchmarked independently of the runtime, and the runtime can swap or
disable the artifact side without downtime.

Within OpenClawBrain, computation splits into two paths:
\begin{itemize}
  \item \textbf{Hot path} (latency-critical): local embedding lookup plus
        bounded graph traversal. No external calls. This is what runs on every
        user-facing query.
  \item \textbf{Cold path} (asynchronous): teacher distillation, policy-gradient
        updates, and structural maintenance. Runs offline; never blocks a
        user-visible request.
\end{itemize}

We define a \emph{turn} as a single LLM API call. Total cost is
\[
C=\sum_{i=1}^{T} \mathrm{tokens}_i \cdot \mathrm{price}_i
\]
OpenClawBrain reduces both $T$ (fewer calls per user-visible request) and
per-call token counts by keeping routing local on the hot path. Teacher calls
happen only in the cold path and never block user-facing latency.

\subsection{What the system learns}
\label{sec:what-learns}

At each routing decision, OpenClawBrain combines two signals over candidate
targets $\mathcal{A}(s)$:
\begin{itemize}
  \item \textbf{Graph prior} $\mathrm{graph\_prior}(a\mid s)$: persistent
        structure learned from past traffic---which nodes tend to follow which.
  \item \textbf{Query-time similarity} $\mathrm{QTsim}(a\mid q,s)$: how well
        each candidate fits the current query, computed from embeddings.
\end{itemize}
Neither signal is sufficient alone. The graph prior captures long-range
sequential patterns that embedding similarity misses; QTsim captures
query-specific relevance that the graph cannot predict from structure alone.

\subsection{How the signals combine}
\label{sec:mixing}

A confidence mixer produces a per-query weight $\lambda \in [0,1]$ from two
features: the \emph{margin} between the top two QTsim scores (high margin means
QTsim is decisive) and the \emph{entropy} of the QTsim distribution (high
entropy means QTsim is uncertain):
\[
\lambda = \sigma\!\big(\alpha_0 + \alpha_1\,\mathrm{margin}_{\mathrm{QTsim}}
  - \alpha_2\,H(\mathrm{QTsim})\big)
\]
\[
\pi(a\mid q,s) = \lambda\,\mathrm{QTsim}(a\mid q,s)
  + (1-\lambda)\,\mathrm{graph\_prior}(a\mid s)
\]
When QTsim is sharp and separated, $\lambda$ rises and the router trusts the
embedding signal. When QTsim is uncertain, $\lambda$ drops and the router falls
back to graph structure. This adaptive fallback is one reason the full system
holds up under sparse feedback, where embedding quality alone is not reliable.

\section{Learning Loop: Distillation + Policy Gradient}
\label{sec:learning}

Think of the learning loop as two gears turning together. The first
gear---\emph{teacher distillation}---gives the brain a fast head start by
imitating an expert. The second gear---\emph{policy-gradient updates}---lets
the brain improve on its own by learning from outcomes. Both gears turn the
same routing function, $\mathrm{route\_fn}$, which mixes a structural graph
prior with a query-time compatibility signal (QTsim).

\subsection{Teacher Distillation (The Fast Gear)}
\label{sec:distillation}

An asynchronous teacher replays recent route decisions offline and labels better
alternatives. This works like a driving instructor reviewing dashcam footage:
the brain never has to stall a live session, but it gets dense
supervision---especially valuable early on, when human corrections are sparse.

The teacher produces soft targets over candidate actions. The learner minimizes
a cross-entropy loss against those targets, pulling $\mathrm{route\_fn}$'s
weights toward the teacher's preferred distribution.

\subsection{Policy-Gradient Update (The Outcome Gear)}
\label{sec:pg}

Once the brain has a reasonable starting policy, outcome-based learning takes
over as the primary driver. Given a route trajectory
$\tau=(s_0,a_0,\dots,s_T,a_T)$ with scalar return $R(\tau)$:
\[
\nabla J(\theta) = \mathbb{E}_{\tau\sim\pi_\theta}\!\left[\sum_{t=0}^{T}
  \bigl(R(\tau)-b_t\bigr)\,\nabla_\theta \log \pi_\theta(a_t\mid s_t)\right]
\]
where $b_t$ is a per-step baseline that reduces variance. In plain terms:
routes that led to good outcomes get reinforced; routes that led to poor
outcomes get weakened. The gradient flows through the same $\mathrm{route\_fn}$
parameters that control the QTsim/graph-prior confidence mix~$\lambda$.

The ablation ladder confirms this design. Across every proof family the
ordering is consistent and never reverses:
\[
\mathrm{route\_fn\_only} \;<\; \mathrm{graph\_route\_pg} \;<\;
\mathrm{full\_brain}
\]
Adding policy-gradient updates on top of the base $\mathrm{route\_fn}$ yields
+11.8 to +26.3~pp; adding structural plasticity (Section~\ref{sec:structure})
on top of that yields another +20.8 to +42.6~pp.

\subsection{Reward-Source Hierarchy}
\label{sec:reward}

Different supervision signals carry different authority. The system resolves
conflicts by an explicit ordering:
\begin{enumerate}
  \item \textbf{Human corrections} (highest authority)---a user saying ``no,
        route to X'' always wins.
  \item \textbf{Self-learning outcomes}---did the chosen route actually help the
        downstream agent?
  \item \textbf{Harvested weak signals}---implicit feedback such as follow-up
        patterns.
  \item \textbf{Teacher labels} (lowest per-sample authority, highest
        volume)---bulk coverage from async replay.
\end{enumerate}
This ordering ensures that a single human correction can override thousands of
teacher labels, while teacher labels still fill the gaps where human feedback is
absent.

\section{RL-Native Maintenance (Structure Learning)}
\label{sec:structure}

A static graph cannot keep up with a world that changes. OpenClawBrain treats
graph maintenance as an ongoing control problem---more like a living nervous
system that grows new connections and prunes unused ones than a database that
waits for a human to reorganize it.

The maintenance pipeline runs in the cold path and follows a fixed sequence on
each cycle:
\[
\text{health} \;\to\; \text{decay} \;\to\; \text{scale} \;\to\; \text{split}
\;\to\; \text{merge} \;\to\; \text{prune} \;\to\; \text{connect}
\]

\subsection{Structural Plasticity: The Largest Contributor}
\label{sec:plasticity}

Structural plasticity---the ability to connect, split, merge, and prune graph
nodes---is the single largest contributor to accuracy in every proof family
tested. Without plasticity the graph slowly drifts out of alignment with the
real world; with it, the brain reshapes itself to match.

The ablation data is stark: plasticity adds +20.8 to +42.6~pp over the
policy-gradient-only stage across all families (Table~\ref{tab:ablation} in
Section~\ref{sec:ablation}).

\subsection{How the Actions Work}
\label{sec:actions}

Think of the graph like a city road map that re-draws itself:
\begin{itemize}
  \item \textbf{Connect:} when the brain repeatedly routes through an indirect
        path successfully, it paves a direct road---creating or strengthening an
        edge that reflects the learned shortcut.
  \item \textbf{Split:} when one node is doing double duty (like a single highway
        exit serving two towns), the brain splits it into specialized nodes so
        each route can be scored independently.
  \item \textbf{Merge:} when two nodes carry redundant content and behave
        identically downstream, they collapse into one, reducing noise in the
        graph.
  \item \textbf{Prune:} edges and nodes that have decayed below a Hebbian
        co-firing threshold are removed, keeping the graph lean.
  \item \textbf{Decay:} connection weights decrease over time unless reinforced
        by successful use---the ``use it or lose it'' principle.
\end{itemize}

Every structural action runs through a safe-apply gate: candidates are scored
using policy-derived signals, and the change is applied only when guardrails
pass. If a structural edit degrades routing performance on replay, it is rolled
back.

\section{Evaluation: Evidence Ladder and Comparative Results}
\label{sec:eval}

Evidence in this section comes from the Brain-vs-RAG Ground-Zero benchmark,
a purpose-built harness that evaluates the full-brain retrieval-routing mechanism
against eight baselines on three distinct task families, each at 10-seed scale.
Two evaluation modes are used and must not be conflated:

\begin{itemize}
  \item \textbf{Recorded head-to-head (H2H):} deterministic fixture replay with
        SHA-256 hash verification. Fully reproducible and the strongest evidence
        tier. Used for the relational-drift family.
  \item \textbf{Simulation:} live simulation with the harness policy function,
        deterministic given a seed. Used for recurring-workflows and
        sparse-feedback families (and additionally for relational-drift
        ablations).
\end{itemize}

\subsection{Proof Boundary}
\label{sec:proof-boundary}

\noindent\textbf{Proven.}
The full-brain mechanism---graph memory, learned route function, policy-gradient
updates, and structural plasticity---dominates the best RAG baseline and all
partial-brain ablations on three long-lived memory tasks at 10-seed scale with
8~baselines, while using 5$\times$ less context per query.
Cross-family head-to-head record: 29-1-0 across 30 seed-level comparisons.

\noindent\textbf{Not claimed.}
Performance on untested families (e.g., memory compaction); behavior at larger
world sizes (current: 50 entities, 80 workflows); end-to-end performance with
live LLM routing (the harness uses simulated policy functions); production
latency or cost; shadow-traffic or online superiority.

\noindent\textbf{Rule.}
Benchmark wins are benchmark claims. Shadow and online production claims remain
out of scope until their artifacts are published.

\label{sec:cross-family}

\noindent\textbf{Relational drift (recorded H2H).}
10~seeds $\times$ 800~queries per seed $\times$ 8~baselines.
Deterministic fixture replay with hash verification.
\texttt{full\_brain} achieves 99.15\% accuracy ($\pm\,0.27$\%) against
89.44\% ($\pm\,1.68$\%) for the best RAG baseline, a margin of +9.71~pp
and a perfect 10-0-0 seed record (Figure~\ref{fig:recorded-h2h}).

\noindent\textbf{Recurring workflows (simulation).}
10~seeds $\times$ 3{,}522~queries per seed $\times$ 8~baselines.
\texttt{full\_brain} reaches 97.6\% ($\pm\,0.4$\%) vs.\ 70.6\% ($\pm\,1.2$\%)
for the best RAG baseline, a margin of +26.9~pp and a 10-0-0 record.
This is the largest margin observed across families.

\noindent\textbf{Sparse feedback (simulation).}
10~seeds $\times$ 1{,}800~queries per seed $\times$ 8~baselines.
Explicit feedback covers approximately 19\% of queries.
\texttt{full\_brain} reaches 92.0\% ($\pm\,18.3$\%) vs.\ 67.0\% ($\pm\,1.4$\%),
a margin of +24.9~pp.
The high standard deviation ($\pm\,18.3$\%) reflects seed-level variance under
sparse supervision: accuracy depends on which subset of queries receives explicit
feedback. The head-to-head record is 9-1-0, preventing an unqualified
``always wins'' claim across families.

\subsection{Context Efficiency}
\label{sec:context}

Across all three families, \texttt{full\_brain} uses 1~context unit per query
vs.\ 5 for \texttt{vector\_rag\_rerank}---a consistent 5$\times$ context
advantage. In the relational-drift family this translates to 800 vs.\
4{,}000 context tokens per query (Figure~\ref{fig:margin-context}, right panel).

\subsection{Baselines and Fairness}
\label{sec:baselines}

Each family runs 8~baselines under identical conditions: same world state, entity
set, query stream order, context budget, correction stream, and scoring rubric.
The baseline set includes:

\begin{itemize}
  \item \texttt{oracle} (ceiling; always correct)
  \item \texttt{full\_brain} (graph memory + learned route + PG + plasticity)
  \item \texttt{graph\_route\_pg} (ablation: no structural plasticity)
  \item \texttt{route\_fn\_only} (ablation: no graph, no PG)
  \item \texttt{vector\_rag\_rerank} (best RAG baseline; top-k + reranker)
  \item \texttt{vector\_rag} (top-k embedding retrieval, no reranking)
  \item \texttt{heuristic\_stateful} (deterministic heuristic with state tracking)
  \item Additional baselines as defined per family
\end{itemize}

\noindent All results are deterministic given a seed. Baselines that inherently
require more context (e.g., \texttt{vector\_rag\_rerank} at 5$\times$) are
measured but not artificially capped.

\section{Operational Notes}
\label{sec:ops}

OpenClawBrain supports local embeddings out-of-the-box for operator-first
deployment. This enables deterministic startup and private/offline-friendly
operation while keeping optional teacher workflows in asynchronous jobs.

All proof-scale evaluations use simulated policy functions with deterministic
outcomes given a seed, not live LLM calls. The benchmark therefore validates the
retrieval-routing mechanism in isolation. End-to-end performance with live model
inference is not measured at proof scale; that rung belongs to shadow-traffic
evaluation (Section~\ref{sec:proof-boundary}).

All baselines in every run receive the same world state, query stream, context
budget, correction stream, and scoring rubric. Context cost is measured, not
estimated: \texttt{full\_brain} uses 1 context unit per query versus 5 for
\texttt{vector\_rag\_rerank} across all families.

\section{Conclusion}
\label{sec:conclusion}

The full-brain mechanism---graph memory with structural plasticity, learned
routing with QTsim confidence mixing, and policy-gradient learning under a
reward-source hierarchy---achieves +9.7 to +26.9~pp over the best RAG baseline
across three proof-scale benchmark families, while using 5$\times$ less context
per query. The cross-family head-to-head record is 29-1-0 across 30 seed-level
comparisons with 8~baselines per family.

The ablation ladder holds without reversal in every family: structural plasticity
is the single largest contributor, adding +20.8 to +42.6~pp over the
policy-gradient-only stage. This consistency across families with different
failure modes---relational drift, recurring workflows, and sparse teacher
feedback---suggests the mechanism is not overfit to a single task structure.

These results are benchmark claims, not production claims. The honest gaps are
stated in Section~\ref{sec:limitations}. Shadow-traffic and online evaluation
remain the next rungs on the evidence ladder.

\subsection{Limitations and Honest Gaps}
\label{sec:limitations}

\textbf{Simulated policy functions.} All proof-scale families use simulated
policy functions with deterministic outcomes, not live LLM calls. The benchmark
validates the retrieval-routing mechanism in isolation; end-to-end performance
with a live model in the loop is unmeasured.

\textbf{Moderate world sizes.} The largest family uses 80~workflows and
3{,}522~queries per seed. Scaling behavior at 10$\times$--100$\times$ is
unknown. Graph traversal cost, plasticity convergence, and homeostatic stability
at larger scales are untested.

\textbf{Seed sensitivity under sparse feedback.} The sparse-feedback family
reports $\pm\,18.3$\% standard deviation across seeds, reflecting high variance
when only $\sim$19\% of queries receive explicit feedback. One seed loss
(9-1-0 head-to-head) prevents an unqualified ``always wins'' claim across
families.

\textbf{Memory compaction not evaluated.} A memory-compaction stress-test family
was designed but not run. The system's behavior under sustained structural churn
is therefore unproven.

\textbf{Custom benchmark.} Ground Zero is purpose-built to test specific failure
modes (relational drift, workflow recurrence, sparse signal). It does not
establish generality on community benchmarks, and results do not directly compare
to other systems on shared evaluation sets.

\textbf{No ablation of teacher supervision.} The teacher signal is part of the
full system but is not ablated independently. The relative contribution of
teacher distillation versus outcome-only learning is not isolated.

\textbf{Embedding dependency.} Retrieval quality is bounded by embedding quality
at the seeding stage. If the correct node is never surfaced by embedding
similarity, the system cannot learn a route to it.

\textbf{No production latency or cost claims.} Latency and cost under production
load are not measured at proof scale.
